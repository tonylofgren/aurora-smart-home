# Matter Bridge

Guide for exposing ESPHome devices to the Matter ecosystem.

## Overview

Matter is a universal smart home standard supported by Apple, Google, Amazon and Samsung. With ESPHome you can:

- Expose ESPHome devices directly to Matter
- Use ESP32 as a Matter bridge
- Control devices via Apple Home, Google Home, Amazon Alexa simultaneously

## Hardware Requirements

| Chip | Matter Support | Thread | Recommendation |
|------|----------------|--------|----------------|
| ESP32 | ✅ WiFi | ❌ | Budget, WiFi only |
| ESP32-S3 | ✅ WiFi | ❌ | More memory |
| ESP32-C3 | ✅ WiFi | ❌ | Compact |
| ESP32-C6 | ✅ WiFi + Thread | ✅ | Best for Matter |
| ESP32-H2 | ❌ (no WiFi) | ✅ | Thread only |

**ESP32-C6 is recommended** for Matter as it supports both WiFi and Thread.

## Enable Matter

```yaml
# Generated by aurora@aurora-smart-home (esphome skill) v1.7.9
# https://github.com/tonylofgren/aurora-smart-home

esphome:
  name: matter-light
  friendly_name: "Matter Lampa"

esp32:
  board: esp32-c6-devkitc-1
  framework:
    type: esp-idf
    version: "5.1.2"
    sdkconfig_options:
      CONFIG_ENABLE_MATTER: y

wifi:
  ssid: !secret wifi_ssid
  password: !secret wifi_password

# Matter is enabled here
matter:
  # Device type - see list below
  device_type: light

api:
  # API is still needed for ESPHome Dashboard

ota:
  platform: esphome

logger:
```

## Matter Device Types

| Device Type | Description | Entities |
|-------------|-------------|-----------|
| `light` | Light | light |
| `dimmable_light` | Dimmable light | light (brightness) |
| `color_temperature_light` | Warm white-cool white | light (color_temp) |
| `extended_color_light` | RGB/RGBW | light (rgb) |
| `on_off_plug` | Simple switch | switch |
| `dimmable_plug` | Dimmable outlet | switch (brightness) |
| `temperature_sensor` | Temperature | sensor |
| `humidity_sensor` | Humidity | sensor |
| `contact_sensor` | Door/window | binary_sensor |
| `occupancy_sensor` | Motion | binary_sensor |
| `lock` | Lock | lock |
| `thermostat` | Thermostat | climate |
| `fan` | Fan | fan |
| `cover` | Curtain/blind | cover |

```yaml

  friendly_name: "Matter Glödlampa"

  framework:
    type: esp-idf

  password: !secret wifi_password

  device_type: dimmable_light

  platform: esphome

  - platform: ledc
    pin: GPIO8
    id: light_output

  - platform: monochromatic
    name: "Lampa"
    output: light_output
    # This light is automatically exposed to Matter
```

    version: "5.1.2"
    sdkconfig_options:
      CONFIG_OPENTHREAD_ENABLED: y
      CONFIG_ENABLE_MATTER: y

  # Thread credentials are obtained from the Thread Border Router

  device_type: dimmable_light

## Pairing Process

### QR Code Generation

ESPHome automatically generates a QR code during compilation. It is displayed in:
1. ESPHome Dashboard logs
2. The device's web interface

### Manual Pairing Code

```yaml
matter:
  device_type: light
  # Custom pairing code (optional)
  discriminator: 3840
  passcode: 20202021
```

### Pairing via Apple Home

1. Open the Apple Home app
2. Tap "+" - "Add Accessory"
3. Scan the QR code from ESPHome

### Pairing via Google Home

1. Open the Google Home app
2. "+" - "Set up device" - "Works with Google"
3. Select "Matter" and scan the QR code

### Pairing via Amazon Alexa

1. Open the Alexa app
2. "Devices" - "+" - "Add Device"
3. Select "Matter" and scan the QR code

## Multi-Admin (Multi-Fabric)

Matter supports pairing the same device with multiple ecosystems:

```
┌─────────────────────────────────────────────┐
│           ESPHome Matter Device              │
├─────────────────────────────────────────────┤
│                                              │
│  ┌───────────┐ ┌───────────┐ ┌───────────┐  │
│  │Apple Home │ │Google Home│ │Amazon Echo│  │
│  │  Fabric   │ │  Fabric   │ │  Fabric   │  │
│  └───────────┘ └───────────┘ └───────────┘  │
│                                              │
└─────────────────────────────────────────────┘
```

To pair with additional ecosystems:
1. Open the first ecosystem's app
2. Find "Share device" or "Turn on pairing mode"
3. Scan with the next ecosystem's app

## Limitations

| Limitation | Description |
|-------------|-------------|
| One device_type | Each ESPHome device can only have one Matter device type |
| Memory | Matter requires a lot of flash/RAM - use ESP32-C6 or better |
| Thread | Only ESP32-C6 and ESP32-H2 support Thread |
| Beta | ESPHome Matter is still experimental |

## Troubleshooting

### Device not visible

1. **Check logs** - Matter pairing info is shown in ESPHome logs
2. **Same network** - Phone and ESP must be on the same WiFi
3. **Router multicast** - Some routers block mDNS/multicast

### Pairing fails

1. **Factory reset** - Remove and re-add in ESPHome
2. **Update firmware** - Latest ESPHome version
3. **Check QR code** - Generate a new one if the old one is outdated

### Unstable connection

```yaml
wifi:
  ssid: !secret wifi_ssid
  password: !secret wifi_password
  # Fast connect for stability
  fast_connect: true
  # Static IP is recommended
  manual_ip:
    static_ip: 192.168.1.100
    gateway: 192.168.1.1
    subnet: 255.255.255.0
```

## Zigbee Platform (since 2026.2)

ESPHome's Zigbee support has been expanded in 2026.2 with more device types supported directly. ESP32-C6 and ESP32-H2 can function as Zigbee end devices.

### Basic Zigbee Configuration

```yaml
esp32:
  board: esp32-c6-devkitm-1
  framework:
    type: esp-idf

zigbee:
  # Device acts as Zigbee end device
  role: end_device

light:
  - platform: binary
    name: "Zigbee Light"
    output: relay_output
    # Automatically exposed via Zigbee

sensor:
  - platform: dht
    pin: GPIO4
    temperature:
      name: "Temperature"
    humidity:
      name: "Humidity"
    # Sensors are exposed as Zigbee clusters
```

### Zigbee vs Matter-over-Thread

| Feature | Zigbee | Matter-over-Thread |
|---------|--------|-------------------|
| Ecosystem | ZHA, Zigbee2MQTT | Apple/Google/Amazon |
| Protocol | Zigbee 3.0 | Matter 1.x |
| Chips | ESP32-C6, H2 | ESP32-C6, H2 |
| Mesh | Yes (via router) | Yes (via Thread) |
| ESPHome maturity | New (2026.2+) | Newer but active |

### Important Notes

- Zigbee and WiFi **cannot** run simultaneously on ESP32-C6 (shared radio)
- ESP32-H2 is best for Zigbee/Thread (dedicated 802.15.4 radio)
- Zigbee support is under active development - check ESPHome changelog for latest status
