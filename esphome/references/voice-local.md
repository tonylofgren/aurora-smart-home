# Local Voice Assistant

Guide for building a fully local voice assistant with ESPHome and Home Assistant.

## Overview

A local voice assistant requires no cloud services and works even without internet. All components run locally:

- **Wake Word** - Micro Wake Word (ESP32-S3) or openWakeWord (HA)
- **Speech-to-Text** - Whisper or faster-whisper
- **Intent Recognition** - Home Assistant Assist
- **Text-to-Speech** - Piper

## Hardware Requirements

### Minimum Requirements (Wake Word on HA)

| Component | Requirement |
|-----------|-------------|
| ESP32 | Any variant |
| Microphone | I2S INMP441 or PDM |
| Speaker | I2S MAX98357A |

### Recommended (Wake Word on device)

| Component | Requirement |
|-----------|-------------|
| ESP32-S3 | PSRAM required |
| Microphone | I2S INMP441 (best) |
| Speaker | I2S MAX98357A + 3W speaker |
| Extra | AMP enable pin for silent pauses |

## Wiring Diagram

### INMP441 Microphone (I2S)

```
INMP441         ESP32-S3
--------        --------
VDD     ───────  3.3V
GND     ───────  GND
SD      ───────  GPIO16 (data)
WS      ───────  GPIO17 (word select/LRCLK)
SCK     ───────  GPIO18 (clock)
L/R     ───────  GND (left channel)
```

### MAX98357A Amplifier (I2S)

```
MAX98357A       ESP32-S3
---------       --------
VIN     ───────  5V
GND     ───────  GND
DIN     ───────  GPIO15 (data)
BCLK    ───────  GPIO14 (bit clock)
LRC     ───────  GPIO13 (left/right clock)
SD      ───────  GPIO48 (enable, optional)
GAIN    ───────  (leave unconnected = 9dB)
```

## Minimal Voice Assistant

Wake word is detected by Home Assistant:

```yaml
# Generated by esphome@aurora-smart-home v1.1.0
# https://github.com/tonylofgren/aurora-smart-home

esphome:
  name: voice-assistant-basic
  friendly_name: "Voice Assistant"
  platformio_options:
    board_build.flash_mode: dio

esp32:
  board: esp32-s3-devkitc-1
  framework:
    type: esp-idf

psram:
  mode: octal
  speed: 80MHz

wifi:
  ssid: !secret wifi_ssid
  password: !secret wifi_password

api:

ota:
  platform: esphome

logger:

# I2S Microphone
i2s_audio:
  - id: i2s_in
    i2s_lrclk_pin: GPIO17
    i2s_bclk_pin: GPIO18

  - id: i2s_out
    i2s_lrclk_pin: GPIO13
    i2s_bclk_pin: GPIO14

microphone:
  - platform: i2s_audio
    id: mic
    i2s_audio_id: i2s_in
    adc_type: external
    i2s_din_pin: GPIO16
    pdm: false
    bits_per_sample: 32bit
    channel: left

speaker:
  - platform: i2s_audio
    id: speaker
    i2s_audio_id: i2s_out
    dac_type: external
    i2s_dout_pin: GPIO15
    mode: mono

voice_assistant:
  id: va
  microphone: mic
  speaker: speaker
  use_wake_word: false  # Wake word on HA
  noise_suppression_level: 2
  auto_gain: 31dBFS
  volume_multiplier: 2.0

  on_listening:
    - light.turn_on:
        id: led
        effect: "Listening"

  on_stt_end:
    - light.turn_on:
        id: led
        effect: "Processing"

  on_tts_start:
    - light.turn_on:
        id: led
        effect: "Speaking"

  on_end:
    - light.turn_off: led
    - wait_until:
        not:
          speaker.is_playing:
    - voice_assistant.start:  # Auto-restart

  on_error:
    - light.turn_on:
        id: led
        red: 100%
        green: 0%
        blue: 0%
    - delay: 1s
    - light.turn_off: led

# Status LED
light:
  - platform: esp32_rmt_led_strip
    id: led
    pin: GPIO48
    num_leds: 1
    rmt_channel: 0
    chipset: SK6812
    rgb_order: GRB
    effects:
      - pulse:
          name: "Listening"
          transition_length: 250ms
          update_interval: 250ms
      - pulse:
          name: "Processing"
          transition_length: 100ms
          update_interval: 100ms
      - pulse:
          name: "Speaking"
          transition_length: 500ms
          update_interval: 500ms
```

## With Micro Wake Word (On-Device)

Wake word runs directly on ESP32-S3:

```yaml
# Generated by esphome@aurora-smart-home v1.1.0
# https://github.com/tonylofgren/aurora-smart-home

esphome:
  name: voice-assistant-local
  friendly_name: "Local Voice Assistant"
  platformio_options:
    board_build.flash_mode: dio
    board_build.psram_type: opi

esp32:
  board: esp32-s3-devkitc-1
  flash_size: 16MB
  framework:
    type: esp-idf
    sdkconfig_options:
      CONFIG_ESP32S3_DEFAULT_CPU_FREQ_240: "y"
      CONFIG_ESP32S3_DATA_CACHE_64KB: "y"
      CONFIG_ESP32S3_DATA_CACHE_LINE_64B: "y"

psram:
  mode: octal
  speed: 80MHz

wifi:
  ssid: !secret wifi_ssid
  password: !secret wifi_password

api:
  on_client_connected:
    - voice_assistant.start_continuous:

ota:
  platform: esphome

logger:
  level: DEBUG

i2s_audio:
  - id: i2s_in
    i2s_lrclk_pin: GPIO17
    i2s_bclk_pin: GPIO18
  - id: i2s_out
    i2s_lrclk_pin: GPIO13
    i2s_bclk_pin: GPIO14

microphone:
  - platform: i2s_audio
    id: mic
    i2s_audio_id: i2s_in
    adc_type: external
    i2s_din_pin: GPIO16
    pdm: false
    bits_per_sample: 32bit
    channel: left

speaker:
  - platform: i2s_audio
    id: speaker
    i2s_audio_id: i2s_out
    dac_type: external
    i2s_dout_pin: GPIO15

# Micro Wake Word - runs on ESP32-S3
micro_wake_word:
  models:
    - model: hey_jarvis  # Built-in model
  on_wake_word_detected:
    - voice_assistant.start:
        wake_word: !lambda return wake_word;

voice_assistant:
  id: va
  microphone: mic
  speaker: speaker
  use_wake_word: true
  noise_suppression_level: 2
  auto_gain: 31dBFS
  volume_multiplier: 2.0

  on_listening:
    - light.turn_on:
        id: led
        red: 0%
        green: 0%
        blue: 100%

  on_stt_end:
    - light.turn_on:
        id: led
        red: 100%
        green: 100%
        blue: 0%

  on_tts_start:
    - light.turn_on:
        id: led
        red: 0%
        green: 100%
        blue: 0%

  on_end:
    - light.turn_off: led
    - micro_wake_word.start:  # Listen for wake word again

  on_error:
    - light.turn_on:
        id: led
        red: 100%
        green: 0%
        blue: 0%
    - delay: 1s
    - micro_wake_word.start:

light:
  - platform: esp32_rmt_led_strip
    id: led
    pin: GPIO48
    num_leds: 1
    rmt_channel: 0
    chipset: WS2812
    rgb_order: GRB
```

## Home Assistant Setup

### 1. Install Whisper

```yaml
# HA Add-on: Whisper
# Or in configuration.yaml:
whisper:
```

**Recommended models:**
- `tiny.en` - Fastest, English
- `base` - Balanced
- `small` - Better accuracy

### 2. Install Piper

```yaml
# HA Add-on: Piper
# Voices for Swedish:
# - sv_SE-nst-medium
```

### 3. Configure Assist Pipeline

Settings -> Voice assistants -> Add assistant:
- **Conversation agent:** Home Assistant
- **Speech-to-text:** Whisper
- **Text-to-speech:** Piper
- **Wake word:** (micro wake word if on device)

## With Physical Button

Start voice assistant with a button press:

```yaml
binary_sensor:
  - platform: gpio
    pin:
      number: GPIO0
      mode: INPUT_PULLUP
      inverted: true
    id: button
    on_press:
      - voice_assistant.start:
    on_release:
      - voice_assistant.stop:
```

## With LED Ring (WS2812)

Show status with an LED ring:

```yaml
light:
  - platform: esp32_rmt_led_strip
    id: led_ring
    pin: GPIO48
    num_leds: 12
    rmt_channel: 0
    chipset: WS2812
    rgb_order: GRB
    effects:
      - addressable_rainbow:
          name: "Listening"
          speed: 10
          width: 12
      - addressable_scan:
          name: "Processing"
          move_interval: 50ms
          scan_width: 3
      - addressable_color_wipe:
          name: "Speaking"
          colors:
            - red: 0%
              green: 100%
              blue: 0%
              num_leds: 12
```

## Troubleshooting

### Microphone not working

1. **Check wiring** - I2S is sensitive to bad connections
2. **Check pins** - LRCLK, BCLK, DIN must match
3. **Test with logger:**
```yaml
logger:
  level: VERBOSE
```

### Wake word not detected

1. **PSRAM required** - Micro Wake Word needs ESP32-S3 with PSRAM
2. **Model missing** - Download and specify model
3. **Sensitivity** - Increase `vad_threshold`:
```yaml
micro_wake_word:
  vad_threshold: 1
```

### Poor audio quality

1. **Gain:**
```yaml
voice_assistant:
  auto_gain: 31dBFS
  noise_suppression_level: 2
  volume_multiplier: 3.0
```

2. **Placement** - Avoid echo and background noise
3. **Speaker** - Use at least a 3W speaker

### High latency

1. **Whisper model** - Use `tiny.en` for fastest
2. **Local server** - Run Whisper on a powerful machine
3. **WiFi** - Use 5GHz or static IP
