# Smart Doorbell Camera

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

A self-contained smart doorbell built on the AI-Thinker ESP32-CAM. A press of the tactile button pulses the existing mechanical chime, snaps a 1024x768 photo through the OV2640 sensor, and pushes it to your phone via Home Assistant. A live MJPEG stream is available for Lovelace and any RTSP/WebRTC bridge.

## Features

- Snapshot on press, delivered to mobile devices with the photo attached
- Live preview stream at port 8080 for dashboards and WebRTC
- Optional 5V relay to retrofit an existing mechanical chime
- IR LED output (re-using the AI-Thinker on-board flash LED) for low-light captures
- Local fallback Access Point if WiFi is unreachable
- Brownout-tolerant boot configuration

## Hardware List

| Component | Purpose | Notes |
|-----------|---------|-------|
| ESP32-CAM (AI-Thinker) | MCU + OV2640 camera | The original variant, not the S3 |
| Tactile push button (NO) | Doorbell trigger | Weatherproof IP65 if outdoors |
| 5V relay module (optional) | Pulses mechanical chime | Choose opto-isolated; mounts indoors near the transformer |
| IR LED + 100 ohm resistor (optional) | Night vision fill | Driven from the flash LED rail |
| 5V / 2A power supply | Camera draws spikes up to 600 mA | Cheap 1A units cause brownouts |

The ESP32-CAM does not have a USB port. You need a separate FTDI/USB-TTL adapter (3.3V logic) to flash it the first time, then OTA updates handle the rest.

## Wiring

```
ESP32-CAM (AI-Thinker)
 |
 +-- GPIO12  ---- Doorbell button --+
 |                                  |
 |                                 GND
 |
 +-- GPIO13  ---- Relay IN
 |                Relay COM/NO in parallel with existing chime button
 |
 +-- GPIO4   ---- On-board flash LED (also IR LED if added)
 |
 +-- 5V      ---- Regulated 5V supply (NOT VIN through a long thin cable)
 +-- GND     ---- Common ground
```

GPIO0 and GPIO2 are strapping pins on the ESP32 - keep them clear, especially GPIO0 which must be pulled low only during flashing.

## Installation

1. **Flash the device** with a 3.3V USB-TTL adapter (RX-to-TX, TX-to-RX, GND-to-GND, GPIO0-to-GND during boot only):

   ```bash
   esphome run smart-doorbell-camera.yaml
   ```

2. **Mount the camera** with a clear sightline of the visitor's face. Avoid pointing into direct sun - the OV2640 has narrow dynamic range.

3. **Wire the chime relay** (optional). Connect the relay's NO contacts in parallel with the existing doorbell button. The 0.5 s pulse simulates a press without removing your original wiring.

4. **Create the snapshot folder**:

   ```bash
   mkdir -p /config/www/doorbell
   ```

5. **Add the automations** in `automations.yaml` and reload. Replace `notify.mobile_app_phone` with your own mobile app target.

## Configuration

### Snapshot resolution

`1024x768` (XGA) is the default. Drop to `800x600` if your WiFi is weak; bump to `1600x1200` (UXGA) only if you have strong signal - the upload can take several seconds on lossy links and will block the next ring.

### Live stream

The MJPEG stream is exposed on `http://<device-ip>:8080`. For Lovelace, add it as a Generic Camera or pair with go2rtc / WebRTC Camera for low-latency live view.

### Quiet hours

The TTS announcement automation is gated between 08:00 and 21:30. The chime relay and push notifications fire 24/7 - adjust to taste.

## Thermal and Power Notes

The ESP32-CAM module runs hot under continuous streaming - surface temperatures of 60-70 C are normal. Mount in an enclosure with passive ventilation slots, and keep it out of direct sun. If the chip temperature sensor reports above 80 C consistently, lower `max_framerate` or stop using the live stream when idle.

Brownouts are by far the most common cause of random reboots. The OV2640 pulls short current spikes that collapse the rail on undersized power supplies. Use a regulated 5V/2A brick, keep the cable run short (less than 1.5 m of 22 AWG, or use thicker wire), and add a 470 uF electrolytic close to the 5V pin if you see boot loops.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Boot loops, "Brownout detector triggered" in logs | Upgrade PSU to 5V/2A, shorten cable, add bulk capacitor |
| Camera init fails | Confirm AI-Thinker pinout; some clones swap GPIO22/23 |
| Stream black or pink-tinted | Power cycle; OV2640 init occasionally fails - watchdog recovers it |
| WiFi drops near the door | Reposition router or add a mesh node; ESP32-CAM antenna is weak |
| Snapshot missing in push | Confirm `/config/www/doorbell/` exists and HA can write to it |
