# Battery Temperature & Humidity Sensor

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

A tiny battery-powered ESP32-C3 sensor that wakes every 10 minutes, reads
temperature and humidity from an SHT40, reports battery level, and goes back
to deep sleep - the classic "first ESPHome project" that lasts 6-12 months
on a single charge.

## Features

- **Accurate Climate Readings**: SHT40 sensor with +/- 0.2 C and +/- 1.8% RH accuracy
- **Long Battery Life**: 6-12 months on a 2000 mAh LiPo at a 10-minute interval
- **Battery Monitoring**: Live voltage and percentage via a 1:1 voltage divider
- **OTA Friendly**: Hold the BOOT button on power-up to keep the device awake for firmware updates
- **Compact Hardware**: Fits in a matchbox - Xiao ESP32-C3 is 21 x 17.8 mm
- **Simple to Build**: Six solder joints, no PCB required
- **HA Native**: Auto-discovery via the ESPHome API - no MQTT needed

## Wiring Diagram

```
                  Seeed XIAO ESP32-C3
                  +-----------------+
              5V -|  5V          GND|- GND
             GND -| GND          3V3|- 3V3 ---+
                  |                 |         |
   Battery+ ---+--| GPIO4 (ADC)     |         |
               |  | GPIO5           |         |
               R1 | GPIO6 (SDA)-----+----- SHT40 SDA
               |  | GPIO7 (SCL)-----+----- SHT40 SCL
              GND | GPIO8           |         |
               |  | GPIO9 (BOOT btn)|         |
               R2 | GPIO10          |         |
               |  |                 |         |
              GND +-----------------+         |
                                              |
   SHT40 VDD (pin 5) <-------------------------+
   SHT40 GND (pin 1) <-----------------------> GND
   SHT40 SDA (pin 4) <-----------------------> GPIO6
   SHT40 SCL (pin 6) <-----------------------> GPIO7

   Battery voltage divider:
     Battery+ ---[R1 100k]---+---[R2 100k]--- GND
                             |
                          GPIO4 (ADC)

   Note: 1:1 divider so 4.2 V battery reads as 2.1 V at the ADC,
   safely within the 12 dB attenuation range (~2.45 V max).
```

## Installation

1. **Copy and edit secrets**:

   ```bash
   cp secrets.yaml.example secrets.yaml
   # Edit secrets.yaml with your WiFi credentials and a new API key
   ```

2. **Generate an API encryption key**:

   ```bash
   openssl rand -base64 32
   ```

   Paste the output into `api_encryption_key` in `secrets.yaml`.

3. **Flash the Xiao ESP32-C3 over USB**:

   ```bash
   esphome run battery-temp-sensor.yaml
   ```

   The first flash must be over USB. After that, OTA works over WiFi.

4. **Wire the SHT40**:
   - Connect VDD to 3V3
   - Connect GND to GND
   - Connect SDA to GPIO6
   - Connect SCL to GPIO7
   - Internal pull-ups in the ESP32-C3 are sufficient at 100 kHz

5. **(Optional) Add the battery voltage divider**:
   - Solder two 100 k resistors in series between Battery+ and GND
   - Tap the midpoint to GPIO4
   - This lets the firmware report battery voltage and percentage

6. **Connect the LiPo**:
   - Plug the JST-PH connector into the Xiao's battery port
   - The Xiao charges the battery whenever USB-C is connected

7. **Add the device in Home Assistant**:
   - It should auto-discover via the ESPHome integration
   - Settings > Devices & Services > ESPHome > Configure
   - Paste the same `api_encryption_key`

8. **Add automations and dashboard**:
   - Copy `automations.yaml` contents into Home Assistant's automations
   - Import `dashboard.yaml` via Settings > Dashboards > Add Dashboard > Import

## Configuration

### Sleep Interval

The default is 10 minutes. Change the `sleep_interval` substitution at the top
of `battery-temp-sensor.yaml`:

```yaml
substitutions:
  sleep_interval: "10min"   # Try 5min for testing, 30min for longer battery life
```

Rough battery-life estimates with a 2000 mAh LiPo:

| Interval | Estimated Life |
|----------|----------------|
| 1 min    | 3-4 weeks      |
| 5 min    | 3-4 months     |
| 10 min   | 6-9 months     |
| 30 min   | 12-18 months   |

### Battery Calibration

If your voltage reading is off, adjust the multiplier filter in the YAML:

```yaml
filters:
  - multiply: 2.0   # Increase slightly if reading is low, decrease if high
```

Measure your battery with a multimeter, compare to the reported voltage,
and tune the multiplier until they match.

### Stay-Awake for OTA

Deep-sleep devices are hard to update. To keep the sensor awake:

1. Power-cycle the Xiao while holding the BOOT button (GPIO9), or
2. Toggle the **Prevent Sleep** switch in Home Assistant while the device is awake

The device will stay online until you reboot it.

## Power-Saving Tips

1. **Disable serial logging in production**: `baud_rate: 0` is already set
2. **Use `fast_connect: true`**: Already enabled - reuses stored BSSID
3. **Avoid 2.4 GHz channel hopping**: Pin your router to a fixed channel
4. **Choose 5 GHz only as a last resort**: The C3 only supports 2.4 GHz
5. **Place near the access point**: Weak WiFi forces longer wake times
6. **Static IP cuts ~1 second per wake**: Optional, see ESPHome WiFi docs

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Device flashes once, never reconnects | You forgot to hold BOOT for OTA. Re-flash via USB. |
| Battery dies in days | Check that deep sleep is active - look at `sensor.uptime` to confirm short wake cycles |
| Temperature reads ~5 C high | The Xiao's ESP32 chip warms the SHT40 when nearby. Mount the SHT40 on flying leads, away from the ESP. |
| Battery percentage stuck at 0% or 100% | Voltage divider not wired or `multiply` value wrong - calibrate against a multimeter |
| Sensor "unavailable" in HA | WiFi range issue, or battery below 3.3 V. Plug in USB and check logs. |
| "i2c bus scan: no devices found" | Check SDA/SCL wiring - swap pins, verify 3.3 V to the SHT40 |
| OTA times out | Hold BOOT during power-on, or toggle the Prevent Sleep switch first |

## How Deep Sleep Works Here

The lifecycle of one wake cycle:

1. RTC timer fires after `sleep_interval` (default 10 min)
2. ESP32-C3 boots and connects to WiFi (~1-2 seconds)
3. `on_boot` priority -100 runs after the API client is up
4. SHT40 is read once via `component.update`
5. Battery ADC is read and filtered
6. Values are published to Home Assistant
7. A short delay lets the API confirm receipt
8. `deep_sleep.enter` puts the chip into ULP sleep (~5 uA)

If the stay-awake button is pressed, step 8 is skipped and the device
remains online until the next power cycle.

## Going Further

Once you have this working, easy upgrades:

- **Add a second SHT40** on the same I2C bus (address 0x45 via the SEL pin)
- **Use a solar cell** with a CN3791 MPPT charger for indefinite outdoor use
- **Replace the SHT40 with a BME280** to also get atmospheric pressure
- **Swap the Xiao for an ESP32-C6** for Thread/Matter (requires `esp-idf`)
- **Print a custom enclosure** - search Printables for "Xiao C3 sensor case"
