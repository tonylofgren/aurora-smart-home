# Solar Inverter via Modbus RTU

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

Read live data from a string solar inverter over RS485 Modbus RTU using an
ESP32 and a cheap transceiver module. Reports PV string voltage and power,
AC output, today's yield, lifetime yield, and inverter temperature back to
Home Assistant on a 30 second poll cycle. The energy registers wire straight
into the HA Energy dashboard for daily and lifetime solar tracking.

This recipe defaults to the **Growatt SPF / MIN** series, which is the most
common low-cost single-phase off-grid and hybrid inverter sold in Europe
and Asia. Huawei SUN2000 and Polar variants are covered as alternatives
below.

## Features

- Live PV1 and PV2 string voltage and power
- AC output voltage, frequency, and active power
- Today's energy and lifetime energy as `total_increasing` for the Energy dashboard
- Inverter temperature for derating diagnostics
- Status text (`waiting`, `normal`, `checking`, `fault`) derived from the system register
- 30 second poll interval, light enough to run a year on a USB phone charger
- No cloud dependency, no ShinePhone, no FusionSolar account

## Hardware list

| Component | Purpose | Notes |
|-----------|---------|-------|
| ESP32 dev board | MCU and WiFi | ESP32-WROOM-32 or similar. ESP8266 also works but only has one hardware UART. |
| RS485 transceiver module | TTL <-> RS485 conversion | HW-519 (auto direction) or MAX485 (needs DE/RE pin). Both cost under 2 USD. |
| 2-wire shielded twisted pair | Inverter to ESP | CAT5/6 works for short runs. Belden 9841 or similar for runs over 10 m. |
| Inverter Modbus port plug | Connect to inverter | Growatt uses an RJ45 jack on the side; Huawei uses a 4-pin Phoenix plug. Check your manual. |
| 5 V USB supply for ESP | Power | Any phone charger. |

Total parts cost is around 8 USD if you already have an ESP32 lying around.

## Wiring

```
ESP32                  HW-519 / MAX485                Inverter Modbus port
                       transceiver
+----------+           +------------+                +-------------------+
|     5V --|--- VCC ---|            |                |                   |
|    GND --|--- GND ---|            |                |                   |
|  GPIO17 -|--- DI ----|            |--- A / D+ -----|--- A (RS485+)     |
|  GPIO16 -|--- RO ----|            |--- B / D- -----|--- B (RS485-)     |
|         -|--- DE+RE -| (tied      |                |                   |
|          |    high   | on HW-519, |                |                   |
|          |    or to  | manual on  |                |                   |
|          |    GPIO4) | MAX485)    |                |                   |
+----------+           +------------+                +-------------------+
```

If you use the HW-519 (auto direction), no `flow_control_pin` is needed.
If you use a bare MAX485, tie DE and RE to GPIO4 and uncomment the
`flow_control_pin: GPIO4` line in the YAML.

Use shielded cable for runs over a few meters and ground the shield at
the inverter end only. Long parallel runs next to mains will pick up
noise and corrupt frames.

## Inverter compatibility

This example ships with a Growatt SPF / MIN register map. Other inverters
work with the same hardware, only the register addresses change.

| Inverter | Slave addr | Baud | Parity | Notes |
|----------|------------|------|--------|-------|
| Growatt SPF / MIN / TL3-S | 1 | 9600 | NONE | Default in this recipe |
| Huawei SUN2000 | 1 | 9600 | NONE | Different register map; see below |
| Polar PV-series | 1 | 9600 | EVEN | Use `parity: EVEN` in the UART block |
| SMA Sunny Boy | 3 | 9600 | NONE | Newer firmware only; older needs Speedwire |
| GoodWe DT/MS | 247 | 9600 | NONE | Address 247 is the gateway default |

### Huawei SUN2000

Register layout differs significantly. Use the `huawei_solar` community
component instead of raw Modbus:

```yaml
external_components:
  - source: github://wlcrs/huawei_solar
    components: [huawei_solar]
```

The SUN2000 also speaks Modbus over WiFi via its built-in dongle, so the
RS485 hardware is optional. Pick whichever path matches your install.

### Polar inverters

Polar uses the Growatt register set under the hood but with EVEN parity
and a different scaling on the energy registers. Change two lines in the
YAML:

```yaml
uart:
  parity: EVEN

# In the energy_today / energy_total filters:
filters:
  - multiply: 0.01    # was 0.1 for Growatt
```

## Register map used

All registers below are input registers (function code 04). Multi-byte
values are big-endian `U_DWORD` unless noted.

| Address | Description | Scale | Unit |
|---------|-------------|-------|------|
| 0x0000 | System status (0=waiting, 1=normal, 3=fault) | 1 | code |
| 0x0001 | PV1 voltage | 0.1 | V |
| 0x0003 | PV1 input power (32-bit) | 0.1 | W |
| 0x0009 | PV2 voltage | 0.1 | V |
| 0x000B | PV2 input power (32-bit) | 0.1 | W |
| 0x000D | AC output voltage | 0.1 | V |
| 0x000E | AC frequency | 0.01 | Hz |
| 0x000F | Output power total (32-bit) | 0.1 | W |
| 0x0035 | Energy today (32-bit) | 0.1 | kWh |
| 0x0037 | Energy lifetime (32-bit) | 0.1 | kWh |
| 0x0093 | Inverter temperature | 0.1 | C |

The official Growatt protocol PDF (search "Growatt Modbus RTU Protocol")
lists another 200+ registers covering battery, BMS, and grid statistics.
Add them in the YAML by copying any of the existing sensor blocks and
swapping the address.

## Installation

### 1. Configure secrets

```bash
cp secrets.yaml.example secrets.yaml
# edit secrets.yaml and fill in wifi_ssid, wifi_password, api_key, ota_password
```

### 2. Verify Modbus address and baud rate

On the inverter display, open Settings -> Communications (Growatt) or
Settings -> RS485 (Huawei). Confirm:

- Slave address (default 1)
- Baud rate (default 9600)
- Parity (default NONE for Growatt, EVEN for Polar)

If your installation differs, edit `modbus_address` in the substitutions
block and the `parity` value in the `uart:` block.

### 3. Flash the ESP32

```bash
esphome run solar-inverter-modbus.yaml
```

The first flash must be over USB. Subsequent updates go over WiFi via OTA.

### 4. Wire to the inverter

1. Power down the inverter at the DC disconnect and AC breaker.
2. Open the Modbus port cover.
3. Crimp the RJ45 or 4-pin plug. Pin 1 is A, pin 2 is B on Growatt RJ45.
4. Run the cable to the ESP32 and the transceiver.
5. Power up the inverter, then plug in the ESP32.

### 5. Add to Home Assistant

The device appears automatically in Settings -> Devices -> ESPHome. Add
the encryption key when prompted. All sensors show up in the device page.

### 6. Wire the Energy dashboard

The `sensor.solar_inverter_energy_today` and
`sensor.solar_inverter_energy_total` entities are exposed with
`device_class: energy` and `state_class: total_increasing`, which is
exactly what the HA Energy dashboard wants.

1. Settings -> Dashboards -> Energy.
2. Under **Solar panels**, click **Add solar production**.
3. Pick `sensor.solar_inverter_energy_total` as the lifetime source. If
   the inverter has been running for a while, HA will use the
   accumulated total as the baseline and only show delta from that
   point forward.
4. Optional: under **Individual devices** you can also add
   `sensor.solar_inverter_energy_today` to get a per-day bar chart.

Daily and weekly graphs populate within an hour.

## Troubleshooting

| Issue | Likely cause | Fix |
|-------|-------------|-----|
| All sensors read `unknown` | A and B swapped, or wrong baud / parity | Swap A and B. Confirm baud and parity match the inverter display. |
| Some sensors read 0 forever | Register address wrong for your model | Check the inverter model number against the table above. Polar uses a different scale; SMA uses different addresses. |
| Status code stuck at 0 (waiting) | Inverter is genuinely idle (night, no sun) | Wait for sunrise. If still 0 at noon, check the inverter LCD for an error. |
| Random timeouts in the ESP log | Cable too long or too noisy | Use shielded twisted pair. Add a 120 ohm termination resistor across A/B at the far end. |
| Lifetime energy resets when the ESP reboots | None - the value comes from the inverter, not the ESP | If the inverter itself resets, check its internal battery (Growatt) or replace the comms board. |
| Energy dashboard shows negative values | Sign on `U_DWORD` interpreted wrong | Try `value_type: S_DWORD` for inverters that use signed registers (rare). |
