# E-Paper Weather Station

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

A battery-powered e-paper weather display that pulls current conditions and a
3-day forecast from Home Assistant, draws them onto an 800x480 Waveshare
3.97" panel, then sleeps for an hour. Because e-paper retains the last
image with zero power, a 3000 mAh LiPo realistically lasts 12-18 months.

## Features

- **Beautiful, Always-On Display**: 800x480 e-paper that stays readable at zero draw
- **Hourly Refresh**: Wakes, pulls weather from HA, redraws, sleeps - one cycle uses milliseconds of compute
- **Year-Scale Battery Life**: 12-18 months on a 3000 mAh LiPo
- **Layout**: Current conditions on the left, 3-day forecast on the right, timestamp + battery in the footer
- **Weather Icons**: Material Symbols glyph font, mapped from the HA `weather.*` state string
- **Three Font Sizes**: 96 px temperature, 32 px forecast rows, 18 px footer
- **Force Refresh from HA**: A button entity lets you trigger a redraw whenever you want
- **HA Native**: Zero external services, no MQTT, all data pulled via the ESPHome API

## Wiring Diagram

```
              ESP32-WROOM-32 Dev Board
              +-----------------------+
        5V  - |  5V              GND |- GND -----+
       3V3  - | 3V3                  |           |
              | GPIO35 (ADC) ---------- Battery+ |  via voltage divider
              | GPIO23 (MOSI) ---------- DIN/MOSI
              | GPIO18 (SCK)  ---------- CLK
              | GPIO5  (CS)   ---------- CS
              | GPIO17 (DC)   ---------- DC
              | GPIO16 (RST)  ---------- RST
              | GPIO4  (BUSY) ---------- BUSY
              +-----------------------+
                                            |
        E-Paper 3V3 ------------------------+ from board 3V3
        E-Paper GND ----------------------- GND

   Battery voltage divider:
     Battery+ ---[R1 100k]---+---[R2 100k]--- GND
                             |
                          GPIO35 (ADC1_CH7)
```

The 1:1 divider keeps the ADC well within the 12 dB attenuation range
(roughly 2.45 V at the pin for a 4.2 V battery).

## Installation

1. **Make sure you are on ESPHome 2026.5.0 or newer**:

   The `epaper_spi` platform with `model: ssd1683` was added in 2026.5.0.
   Older releases will fail to compile.

2. **Copy and edit secrets**:

   ```bash
   cp secrets.yaml.example secrets.yaml
   ```

3. **Generate an API encryption key**:

   ```bash
   openssl rand -base64 32
   ```

   Paste the output into `api_encryption_key` in `secrets.yaml`.

4. **Add the template sensors to Home Assistant**:

   Open `automations.yaml` and copy the commented `template:` block at the
   top into your `configuration.yaml` (or a packages file). It creates
   six template sensors that expose `forecast[0..2]` highs and lows from
   `weather.home`. Reload templates after saving.

5. **Confirm the HA entities exist**:

   The display assumes these entity IDs (override the substitutions at the
   top of the YAML if yours differ):

   - `sensor.outdoor_temperature`
   - `sensor.outdoor_humidity`
   - `sensor.wind_speed`
   - `weather.home`
   - `sensor.forecast_today_high`, `sensor.forecast_today_low`
   - `sensor.forecast_tomorrow_high`, `sensor.forecast_tomorrow_low`
   - `sensor.forecast_day3_high`, `sensor.forecast_day3_low`

6. **Flash the ESP32 over USB**:

   ```bash
   esphome run epaper-weather-station.yaml
   ```

7. **Wire the e-paper module** (see diagram above), connect the battery,
   and watch the first refresh complete in 30-60 seconds.

8. **Import the dashboard**: Settings > Dashboards > Add Dashboard > Import.

## Configuration

### Refresh Interval

Hourly is the sweet spot for weather. Change the substitution to tune:

```yaml
substitutions:
  sleep_interval: "60min"   # 30min for high-traffic info, 6h for ultra-long life
```

Rough battery-life estimates on a 3000 mAh LiPo:

| Interval | Estimated Life |
|----------|----------------|
| 15 min   | 4-6 months     |
| 60 min   | 12-18 months   |
| 6 h      | 2-3 years      |

### Switching to the 4.2" Goodisplay Panel

Change one line in `epaper-weather-station.yaml`:

```yaml
display:
  - platform: epaper_spi
    model: "4.20in-bw-ssd1683"   # was: 3.97in-bw-ssd1683
```

Then tighten the layout coordinates in the lambda (400 x 300 instead of
800 x 480). A good starting scale is to halve every `x` and `y` value.

### Custom Weather Icons

The YAML uses Material Symbols Outlined for icons because the font file
is fetched automatically by ESPHome from the Google Fonts CDN. To add
more glyphs, look up codepoints at [fonts.google.com/icons](https://fonts.google.com/icons),
copy the Unicode value, and add it under `glyphs:` in the `font:` block.

For sharper monochrome icons you can also convert SVGs to a custom font
with [fonttools](https://fonttools.readthedocs.io/) and reference the
local TTF via `file: weather-icons.ttf` instead of `gfonts`.

### OTA Updates on a Sleeping Device

Deep-sleep devices are hard to update. Two options:

1. **Software stay-awake**: Toggle the **Prevent Sleep** switch while the
   device is briefly awake (catch the 60-second window after a refresh).
2. **Manual wake**: Press the reset button on the ESP32 board, then call
   the **Refresh Now** button in HA - the device stays online for the full
   `run_duration` (60s) before sleeping again, plenty of time for OTA.

## Home Assistant Integration

### Entities Created

**Sensors**

- `sensor.epaper_weather_battery_voltage` - Battery voltage (V)
- `sensor.epaper_weather_battery` - Battery percentage (0-100)
- `sensor.epaper_weather_wifi` - WiFi RSSI (diagnostic)

**Switches**

- `switch.epaper_weather_prevent_sleep` - Stay awake for OTA

**Buttons**

- `button.epaper_weather_refresh_now` - Force a redraw on the next wake
- `button.epaper_weather_restart` - Reboot the device

**Diagnostic**

- `sensor.epaper_weather_esphome_version` - Firmware version string

## Power-Saving Tips

1. **Baud rate 0**: Already set - serial logging is off in production
2. **`fast_connect: true`**: Stored BSSID skips the WiFi scan (~1s per wake)
3. **Static IP**: Optional, saves another second per wake cycle
4. **Hourly refresh**: Don't go below 15 minutes unless you really need it
5. **Disable the green power LED**: Cut the trace or remove the resistor - some boards waste 5-10 mA on the LED alone
6. **Use a low-quiescent LDO**: If your dev board has an AMS1117, swap it for an HT7333 to drop sleep draw from ~5 mA to ~10 uA

## Troubleshooting

| Issue | Solution |
|-------|----------|
| "Unknown display model ssd1683" | Upgrade ESPHome to 2026.5.0 or newer |
| Display stays white | Check BUSY pin wiring - SSD1683 needs it as input. Also verify 3V3 to the panel. |
| Display shows ghosting | Normal after many partial updates - force a full refresh by power-cycling |
| All forecast rows show "--" | Template sensors not loaded. Confirm `sensor.forecast_today_high` exists in HA. |
| Time shows "1970-01-01" | The `homeassistant` time platform needs the HA API to be connected before the display redraws. Increase the `delay: 8s` to `15s`. |
| Battery dies in days | Confirm deep sleep is active - check that uptime stays low. The green power LED is the most common culprit. |
| OTA times out | Catch the 60-second wake window, toggle Prevent Sleep, then OTA |

## How One Wake Cycle Works

1. RTC timer fires after `sleep_interval` (default 60 min)
2. ESP32 boots and connects to WiFi (~1-2 seconds)
3. `on_boot` priority -100 waits 8 seconds for HA to push fresh sensor values
4. `component.update: epaper` redraws the entire screen via SPI
5. The e-paper holds the new image with zero current
6. After a short delay, `deep_sleep.enter` puts the chip into ULP sleep

## Going Further

- **Use the GxEPD2 partial-refresh mode** for sub-second redraws (loses some battery life)
