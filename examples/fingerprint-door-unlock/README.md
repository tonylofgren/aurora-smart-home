# Fingerprint Door Unlock

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

Optical fingerprint reader paired with an electric strike (or magnetic lock) so authorized fingers unlock the door instantly. Unknown attempts are forwarded to Home Assistant so you can approve a remote unlock from your phone.

## Features

- Sub-second match-to-unlock latency via the on-sensor matcher (no cloud)
- Up to 200 enrolled fingerprints (depending on sensor model)
- HA event for every match and miss, with slot number and confidence
- Remote unlock switch from Home Assistant or mobile actionable notification
- Green/red status LED feedback on the reader
- LiPo backup battery so the lock survives short mains outages
- Battery voltage monitoring with a hard cutoff to prevent latching the door open

## Hardware list

### Controller
- ESP32 DevKit (any 38-pin board) or ESP32-C3 SuperMini

### Reader and lock
| Component | Purpose | Interface |
|-----------|---------|-----------|
| Adafruit GROW R503 (or AS608 / R307) | Optical fingerprint reader | UART 57600 baud |
| 12V electric strike OR magnetic lock | Door release | Dry contact |
| 5V relay module (opto-isolated) | Switches lock power | GPIO |
| Common-cathode RGB LED or 2x discrete LED | Status feedback | PWM |
| 12V 1A power supply | Lock + buck to 5V for ESP | Mains |
| LiPo 18650 + TP4056 + MT3608 boost | Backup power for ESP and lock relay | - |
| 100k / 100k resistor divider | Battery voltage on GPIO34 | ADC |

### Wiring

```
ESP32                       GROW R503
GPIO17 (TX) ─────────────── RX  (white)
GPIO16 (RX) ─────────────── TX  (yellow)
GPIO4              ──────── WAKEUP (blue, touch-detect)
3V3                ──────── VCC red + VT touch-vcc
GND                ──────── GND black

ESP32                       Relay / lock
GPIO5  ──── Relay IN
12V    ──── Relay COM ───── Strike +
Strike -   ──────────────── 12V GND

ESP32                       Status LED
GPIO18 (PWM) ───── 220R ─── Green anode
GPIO19 (PWM) ───── 220R ─── Red anode
GND               ──────── Common cathode

ESP32                       Battery monitor
GPIO34 ── 100k ── VBAT
GPIO34 ── 100k ── GND
```

## Installation

1. Copy `secrets.yaml.example` to `secrets.yaml` and fill in your values.
2. Flash the ESP32:
   ```bash
   esphome run fingerprint-door-unlock.yaml
   ```
3. Wire the relay in series with the strike's power feed. Fail-secure strikes unlock when energized; maglocks unlock when de-energized, so invert the relay's NO/NC contacts if you use a maglock.
4. Mount the reader at door height, route the 6-wire JST cable through the frame, and seal with silicone if exposed to weather.
5. Once the device appears in Home Assistant, copy the automations from `automations.yaml` and the dashboard from `dashboard.yaml`.

## Enrollment workflow

The R503 stores templates on-sensor in slots 0-199. To enroll a new finger:

1. In Home Assistant, open Developer Tools and press one of the enrollment buttons (e.g. `button.front_door_enroll_finger_0`). The dashboard exposes the first two slots; for additional slots, call the `fingerprint_grow.enroll` action via a script or the developer console with the desired `finger_id` (0-199).
2. The reader's status LED turns blue. Place the finger flat on the sensor.
3. Lift and re-place the same finger 5 times. A persistent notification tracks scan progress.
4. On success, the enrolled count sensor increments and a notification confirms the slot.
5. If a scan fails (smudged, partial), press Cancel Enrollment and start again.

Tips:
- Re-enroll thumbs and index fingers separately - they read differently.
- Wipe the platen with a microfibre cloth weekly; oil and dust kill match confidence.
- Confidence below 50 is a marginal match; below 30, re-enroll the slot.

To delete a single slot, use the `fingerprint_grow.delete` action with the matching `finger_id`. To wipe all templates, press Delete All Fingers.

## Security warning

Electric strikes and surface-mount maglocks are a convenience layer, not a primary lock. Common defeats include:

- Shimming the strike's keeper with a credit card.
- Holding a strong magnet against a maglock to confuse the bond sensor.
- Cutting the 12V feed (mains-only installations open on power loss with fail-safe strikes).
- Pulling the door hard against worn frames where the keeper has play.

Mitigations used here:
- A LiPo backup keeps the ESP and relay alive during brief mains drops, so the strike does not chatter or release on a glitch.
- The voltage cutoff (3.4 V default) refuses to fire the strike when the backup is too weak to also drive the lock cleanly; the door stays locked rather than latching open.
- All match and miss events are logged with timestamps and confidence.

Treat this as a way to skip carrying a key, not as a replacement for a real deadbolt. For perimeter security, keep a mechanical deadbolt on the same door and only release the strike for daytime convenience.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Reader never responds | Check TX/RX are crossed, sensor needs 3.3V on both VCC pins |
| Match works in HA log but door does not click | Verify relay wiring and that the strike's 12V supply is healthy |
| LED stays red on every finger | Wipe the platen, re-enroll the slot, drop security level if needed |
| Door clicks but does not release | Wrong polarity for maglock - swap NO/NC contacts on the relay |
| Battery voltage reads 0 | Resistor divider not connected, or ADC pin not on ADC1 (use GPIO32-39) |
| Unknown attempts not notifying | Confirm the automation in `automations.yaml` is enabled and the mobile_app target is correct |
