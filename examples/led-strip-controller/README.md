# LED Strip Controller Example

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

Addressable LED strip controller for WS2812B (or SK6812 RGBW) strips with HA color picker, brightness control, and a rich library of effects. Built on ESPHome's `neopixelbus` platform for ESP32, with mushroom-card dashboard and ready-to-use automations.

## Features

- **Full HA color picker integration**: pick any RGB color from Home Assistant or the Mushroom light tile
- **9 light effects**: Rainbow, Color Wipe, Scan, Twinkle, Random Twinkle, Fireworks, Pulse Slow, Strobe, Boot Wipe
- **Brightness control** via HA slider or automations (gradual fade transitions supported)
- **Boot animation**: short green wipe at startup so you immediately know the controller booted
- **State restore**: remembers off-state on reboot (configurable)
- **Identify button**: flashes the strip white for one second, useful when you have several controllers
- **Parameterized**: change `num_leds` and `data_pin` via substitutions without touching the rest of the config

## Hardware List

### Controller

- ESP32 DevKit (any flavor, ESP32-WROOM-32 recommended) or ESP32-C3 Mini

### LED Strip

| Component | Spec | Notes |
|-----------|------|-------|
| LED strip | WS2812B, 60 LEDs/m | Default config: 60 LEDs (1 m). Adjust `num_leds` to match. |
| Alt: SK6812 RGBW | 4-channel addressable | Change `variant` to `SK6812` and `rgb_order` to `GRBW`. |
| Power supply | 5 V, >= 4 A | See power budget below. |
| Level shifter | 74AHCT125 or 74HCT245 | Strongly recommended (see below). |
| Series resistor | 470 ohm | On data line between level shifter and first LED. |
| Bulk capacitor | 1000 uF, 10 V | Across 5V / GND close to the strip. |
| Fuse | 5 A, slow-blow | Inline with the 5V rail for safety. |

### Power Budget

Each WS2812B LED draws up to 60 mA at full white (R + G + B all on). For 60 LEDs:

```
60 LEDs x 60 mA = 3600 mA = 3.6 A peak
```

Recommended PSU sizing:

| LED count | Peak (full white) | Recommended PSU |
|-----------|-------------------|-----------------|
| 30        | 1.8 A             | 5V / 3 A        |
| 60        | 3.6 A             | 5V / 5 A        |
| 100       | 6.0 A             | 5V / 8 A        |
| 144       | 8.6 A             | 5V / 10 A       |
| 300       | 18.0 A            | Inject power on multiple points, 5V / 20 A |

For runs over 1 m, **inject 5V/GND at both ends** of the strip to avoid voltage drop (a visible red tint at the far end is the symptom).

### Level Shifter (Important)

The ESP32 outputs 3.3 V logic, but WS2812B expects data at 0.7 x VDD = 3.5 V minimum. In practice, 3.3 V data often works, but it is borderline and prone to flicker, color glitches, or the first few LEDs misbehaving.

**Recommended**: a 74AHCT125 or 74HCT245 buffer powered from 5V. Wire it as:

```
ESP32 GPIO13 -> 74AHCT125 input A1
74AHCT125 output Y1 -> 470 ohm -> strip DIN
74AHCT125 VCC -> 5V
74AHCT125 GND -> common GND
74AHCT125 OE (pin 1) -> GND (always enabled)
```

If you do not have a level shifter, keep the data wire under 10 cm and place the strip in series with a small dummy LED (sacrificial first WS2812) that boosts the signal to 5V logic for the rest of the strip.

## Installation

1. **Copy secrets template**:
   ```bash
   cp secrets.yaml.example secrets.yaml
   # then edit secrets.yaml with your WiFi + API key + OTA password
   ```

2. **Adjust substitutions** in `led-strip-controller.yaml` if needed:
   - `num_leds`: total LED count in your strip
   - `data_pin`: GPIO connected to the strip data input
   - `rgb_order`: `GRB` for WS2812B, `GRBW` for SK6812 RGBW
   - `default_brightness`: startup brightness (0-255)

3. **Flash the device**:
   ```bash
   esphome run led-strip-controller.yaml
   ```

4. **Add to Home Assistant**: the device is auto-discovered. Confirm the API encryption key.

5. **Install Mushroom cards** (optional, for the dashboard):
   - HACS -> Frontend -> search for `Mushroom` and install.

6. **Import dashboard**: copy `dashboard.yaml` content into Settings -> Dashboards -> Edit Dashboard -> Raw configuration editor.

7. **Add automations**: copy `automations.yaml` into your `automations.yaml` or import via UI.

## Configuration

### Number of LEDs

Change `num_leds` substitution. ESPHome allocates 3 bytes per LED (4 for RGBW), so 60 LEDs = 180 bytes RAM. Safe up to ~500 LEDs on ESP32. Beyond that, consider splitting into segments or use a faster MCU.

### Different Strip Types

| Strip | `variant` | `rgb_order` |
|-------|-----------|-------------|
| WS2812B | `WS2812` | `GRB` |
| WS2811 (slow) | `WS2811` | `RGB` or `BRG` |
| WS2813 | `WS2813` | `GRB` |
| SK6812 RGB | `SK6812` | `GRB` |
| SK6812 RGBW | `SK6812` | `GRBW` |

Edit `light:` -> `platform: neopixelbus` section in the YAML.

### Custom Effects

The config ships with 9 effects. To add more, see [ESPHome addressable effects docs](https://esphome.io/components/light/index.html#light-effects).

## Home Assistant Integration

### Entities Created

**Light**
- `light.led_strip` - main color + brightness + effect entity

**Sensors**
- `sensor.led_strip_wifi` - WiFi signal (diagnostic)
- `sensor.led_strip_uptime` - uptime (diagnostic)

**Buttons**
- `button.led_strip_identify` - flash white for 1 s
- `button.led_strip_restart` - restart the controller

**Text Sensors**
- `sensor.led_strip_esphome_version`
- `sensor.led_strip_ip`

### Calling Effects from Automations

```yaml
service: light.turn_on
target:
  entity_id: light.led_strip
data:
  effect: "Rainbow"
  brightness: 200
```

Available effect names: `Rainbow`, `Color Wipe`, `Boot Wipe`, `Scan`, `Twinkle`, `Random Twinkle`, `Fireworks`, `Pulse Slow`, `Strobe`. Use `None` to disable effects and return to static color.

### Gradual Fade (Brightness Transition)

Use the `transition` parameter (in seconds):

```yaml
service: light.turn_on
target:
  entity_id: light.led_strip
data:
  brightness: 50
  transition: 30   # fade over 30 seconds
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| First LED is wrong color | Add a level shifter or sacrificial dummy LED. |
| Random flicker | Add 1000 uF cap near strip, add 470 ohm series resistor on data line. |
| Far end is dim or red | Inject 5V/GND at the strip's far end (voltage drop over distance). |
| Strip never lights up | Check `rgb_order` and `variant`. Try `GRB` first. Verify data pin. |
| Colors swapped (red shows as green) | Wrong `rgb_order`. Try `RGB`, `GRB`, `BRG`. |
| ESP32 brownouts | PSU undersized. Add separate 5V supply for the strip; only common GND with ESP32. |
| Effects stutter | Lower `update_interval` or reduce `num_leds`. ESP32-C3 is slower than ESP32. |
| OTA fails | Power draw too high. Disconnect strip during first OTA. |

## Safety Notes

- **Never** power 60+ LEDs from the ESP32 5V pin or USB. Use a dedicated 5V PSU.
- Always tie ESP32 GND, PSU GND, and strip GND together (common ground).
- Add an inline fuse on the 5V rail.
- Long cable runs from PSU to strip should use thick wire (18 AWG or larger for >3 A).

## Extending This Example

- Add **physical buttons** (binary_sensor on GPIO) to toggle effects locally without HA.
- Add a **rotary encoder** for offline brightness control.
- Split the strip into **multiple segments** by defining several `light: platform: partition` entities pointing at slices of the same neopixelbus light.
- Add **music sync** with `addressable_color_wipe` driven by an audio sensor.
