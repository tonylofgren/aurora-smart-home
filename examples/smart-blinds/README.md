# Smart Blinds Example

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

Motorized window blinds with stepper-driven position control, endstop calibration, and Home Assistant integration. Two hardware variants supported in a single YAML: cheap 28BYJ-48 for light shades, or NEMA17 for heavy blackout curtains.

## Features

- Open / close / stop control via Home Assistant or local buttons
- Two endstop switches (open + closed) for safety - motor never stalls at limits
- Position memory survives reboot (NVS-backed)
- Sunrise / sunset automation included
- Heat-protection: auto-close when room overheats
- Jam detection: alerts if endstop is not reached within 2 minutes

## Hardware Variants

### Variant 1 - Light shades (recommended for first build)

| Component | Note |
|-----------|------|
| ESP32 DevKit | Any 30-pin board works |
| 28BYJ-48 stepper | 5 V, geared, very quiet |
| ULN2003 driver | Bundled with the 28BYJ-48 |
| 5 V / 1 A PSU | The motor is the heavy load |
| 2x microswitch | Endstops |
| 3x momentary button | Up / Down / Stop |

Torque: ~30 mNm at the output shaft. Enough for cellular shades, light roller blinds up to ~1.2 m wide. Not enough for heavy blackout curtains.

### Variant 2 - Blackout curtains / heavy loads

| Component | Note |
|-----------|------|
| ESP32 DevKit | |
| NEMA17 stepper | 1.5-2 A per phase typical |
| A4988 driver | Set Vref to motor rating (see below) |
| 12 V / 2 A PSU | Or 24 V for higher torque |
| 100 uF capacitor | Across A4988 VMOT / GND |
| 2x microswitch | Endstops |
| 3x momentary button | Up / Down / Stop |

Set the A4988 current limit with a multimeter: Vref = Imax * 8 * Rcs (typical Rcs is 0.1 ohm, so Vref ~= Imax * 0.8). Start at 0.4 V and increase only if the motor stalls.

## Wiring

```
ESP32 DevKit
- GPIO32 -- Endstop OPEN  (microswitch to GND)
- GPIO33 -- Endstop CLOSE (microswitch to GND)
- GPIO25 -- Button UP     (to GND)
- GPIO26 -- Button DOWN   (to GND)
- GPIO27 -- Button STOP   (to GND)

Variant 1 (ULN2003):
- GPIO14 -- IN1
- GPIO12 -- IN2
- GPIO13 -- IN3
- GPIO15 -- IN4
- 5 V    -- VCC (separate PSU recommended)
- GND    -- common with ESP32

Variant 2 (A4988):
- GPIO18 -- STEP
- GPIO19 -- DIR
- GPIO21 -- ENABLE (active low)
- VMOT   -- 12 V via 100 uF cap to GND
- VDD    -- 3.3 V from ESP32
- GND    -- common
- Tie SLEEP and RESET together (high)
```

All endstops and buttons use internal pull-ups and connect to GND when active. No external resistors needed.

## Installation

1. Copy `secrets.yaml.example` to `secrets.yaml` and fill in your WiFi and keys.
2. Choose your motor variant in `smart-blinds.yaml`:
   - For 28BYJ-48: keep the `stepper.platform: uln2003` block as-is.
   - For NEMA17: comment out the ULN2003 block and uncomment the A4988 block (already present, commented).
3. Flash the ESP32:
   ```bash
   esphome run smart-blinds.yaml
   ```
4. Mount endstops at the physical open and closed positions. Both endstops MUST trigger before the motor reaches its hard mechanical limit, otherwise the motor will stall.
5. Wire buttons and motor according to the diagram above.
6. Calibrate (see next section).

## Calibration

The cover uses a `time_based` platform. It tracks position by how long the motor has been running, so the open/close durations must match reality.

### Procedure

1. After flashing, the cover state is unknown. Press the physical **Down** button (or call `cover.close_cover`). The motor runs until the closed endstop triggers and stops.
2. Now in fully closed state, start a stopwatch and call `cover.open_cover`. Note the time when the open endstop triggers.
3. That time, in seconds, is your `open_duration`. Edit `smart-blinds.yaml`:
   ```yaml
   substitutions:
     open_duration:  "20s"
     close_duration: "20s"
   ```
4. Repeat the timing for closing (it can differ slightly due to gravity or friction).
5. Re-flash. Home Assistant will now show accurate position percentages between 0 and 100.

Typical values: 28BYJ-48 with 50 mm shaft = 15-30 s per metre of blind travel. NEMA17 at 800 steps/s = 5-15 s per metre depending on pulley diameter.

### Recalibration

Recalibrate after any mechanical change (new motor, new pulley, repositioned endstops). The included `button.blinds_calibrate` runs the close-direction calibration helper.

## Safety Notes

- Endstop switches are mandatory. Running a stepper into a mechanical hard stop heats the driver and can demagnetize the rotor.
- Always wire `restore_mode: ALWAYS_OFF` style behaviour: the cover starts in unknown state after a power loss. The first close action drives to the closed endstop and re-syncs position.
- Keep a manual override (a way to disengage the motor from the shaft) for first-time installs. You will want it.
- Use a fused PSU. A jammed stepper can pull continuous full-rated current.

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Motor hums but does not turn | Wrong coil order on ULN2003. Try swapping `pin_b` and `pin_c`. |
| Motor runs wrong direction | Swap `pin_a` and `pin_d`, or for A4988 invert `dir_pin`. |
| Cover position drifts over time | Recalibrate; check belt tension and pulley grub screws. |
| Endstop never triggers | Test the GPIO with a multimeter to GND; check `INPUT_PULLUP` and `inverted: true` match your wiring. |
| A4988 gets very hot | Lower Vref or add a heatsink + active cooling. |
| 28BYJ-48 too weak | Switch to Variant 2 (NEMA17 + A4988). |

## Customisation

- **Tilt slats** (for venetian blinds): add a second `stepper` and `cover` with shorter durations.
- **Multiple windows**: duplicate the substitutions and entities, or flash one ESP32 per window and group them as a `cover_group` in Home Assistant.
- **Solar charging**: replace the PSU with a small LiFePO4 + solar panel. The 28BYJ-48 sleeps between moves (`sleep_when_done: true`) so daily idle draw is sub-mA.
