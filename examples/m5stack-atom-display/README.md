# M5Stack ATOM-S3 Info Display

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

A pocket-sized Home Assistant status panel built on the M5Stack ATOM-S3: a single USB-C cable, a built-in 0.85" 128x128 IPS color LCD, and one front button that cycles four rotating pages of live data from Home Assistant.

## Features

- **Always-on color display**: 0.85" 128x128 IPS panel driven by the GC9A01A controller over SPI
- **Four rotating pages** cycled by the front button (GPIO41):
  - Page 0: Clock, date, outdoor temperature, weather state
  - Page 1: Living room, bedroom, and kitchen light status
  - Page 2: Living room temperature, humidity, and CO2
  - Page 3: Presence for Tony and Anna, plus front door and balcony window state
- **Page indicator dots** at the bottom so you always know which screen you are on
- **Color-coded values**: green for safe, amber for warm, red for open doors or high CO2
- **Backlight as a HA light entity**: dim at night, restore in the morning, or toggle on demand
- **No cloud dependency**: all data flows over the native ESPHome API
- **Powered by USB-C**: 5 V from any phone charger keeps it running indefinitely
- **Tiny footprint**: the ATOM-S3 is 24 x 24 mm, fits anywhere on a desk or shelf

## Wiring Diagram

```
                M5Stack ATOM-S3
                +----------------+
                |                |
                |   +--------+   |
                |   |        |   |   <- 0.85" 128x128 IPS LCD
                |   |  GUI   |   |      GC9A01A driver
                |   |        |   |      SPI: CLK=GPIO15, MOSI=GPIO21
                |   +--------+   |      CS=GPIO14, DC=GPIO42, RST=GPIO48
                |                |      Backlight=GPIO16
                |     [BTN]      |   <- Front button on GPIO41 (active low)
                |                |
                |     USB-C      |   <- Power + flashing
                +-------||-------+
                        ||
                   USB-C cable
                        ||
                   5 V supply

   Page button (GPIO41) - press once to advance to the next page.
   Backlight (GPIO16)   - on by default; exposed as light.display_backlight.
```

All pins are internal to the ATOM-S3 module. There is nothing to solder.

## Installation

1. **Clone or download this example folder**:

   ```bash
   git clone https://github.com/tonylofgren/aurora-smart-home.git
   cd aurora-smart-home/examples/m5stack-atom-display
   ```

2. **Download the Roboto fonts**:

   ```bash
   mkdir fonts
   # Download Roboto-Regular.ttf, Roboto-Medium.ttf, Roboto-Bold.ttf
   # from https://fonts.google.com/specimen/Roboto into ./fonts/
   ```

3. **Copy and edit secrets**:

   ```bash
   cp secrets.yaml.example secrets.yaml
   # edit secrets.yaml and fill in wifi_ssid, wifi_password, api_encryption_key, ota_password, ap_password
   ```

4. **Generate an API encryption key**:

   ```bash
   openssl rand -base64 32
   ```

   Paste the output into `api_encryption_key` in `secrets.yaml`.

5. **Adjust the Home Assistant entity IDs** in `m5stack-atom-display.yaml` to match your own setup. Look for these entries and change them to your actual entities:

   - `weather.home`
   - `sensor.outdoor_temp`
   - `light.living_room`, `light.bedroom`, `light.kitchen`
   - `sensor.living_room_temp`, `sensor.living_room_humidity`, `sensor.living_room_co2`
   - `person.tony`, `person.anna`
   - `binary_sensor.front_door`, `binary_sensor.balcony_window`

6. **Flash the ATOM-S3 over USB-C**:

   ```bash
   esphome run m5stack-atom-display.yaml
   ```

   The first flash must be over USB. After that, OTA works over WiFi.

7. **Add the device in Home Assistant**:

   - It should auto-discover via the ESPHome integration
   - Settings > Devices & Services > ESPHome > Configure
   - Paste the same `api_encryption_key`

8. **Add automations and dashboard** (optional):

   - Copy the contents of `automations.yaml` into Home Assistant
   - Add the card from `dashboard.yaml` to a Lovelace view

## Configuration

### Changing the entities shown

The display reads from Home Assistant via `homeassistant`-platform sensors, binary sensors, and text sensors declared at the top of `m5stack-atom-display.yaml`. To replace any of them:

```yaml
sensor:
  - platform: homeassistant
    id: outdoor_temp
    entity_id: sensor.YOUR_OUTDOOR_TEMP    # change this line
```

The lambda block references entities by their `id:` (for example `id(outdoor_temp).state`), so the rest of the YAML does not need to change.

### Adding more pages

Pages are switched by the global `current_page` integer. The button increments it modulo 4. To add a fifth page:

1. Change the modulo in the `on_press` action from `% 4` to `% 5`.
2. Add an `else if (page == 4) { ... }` branch in the display lambda.
3. Bump the indicator-dot loop from `i < 4` to `i < 5` and adjust the starting x offset.

### Backlight behavior

By default the backlight is `restore_mode: ALWAYS_ON`. To make it remember the last state across reboots, change to `RESTORE_DEFAULT_ON`. To dim at night automatically, see the `automations.yaml` examples.

### Update interval

The display refreshes every 5 seconds (`update_interval: 5s`). For a more responsive button feel the lambda also forces an immediate update on press. If you want a faster automatic refresh, lower the interval - but note that ESP32-S3 SPI updates of the full 128x128 panel are cheap, so 1s is fine.

### Rotation

The default `rotation: 0` matches the ATOM-S3 with the button at the bottom and USB-C on the right. If you mount the device differently, try `rotation: 90`, `180`, or `270`.

## Home Assistant Integration

### Entities Created

**Light**

- `light.atom_s3_display_display_backlight` - GPIO16 backlight, on/off

**Binary sensors**

- `binary_sensor.atom_s3_display_page_button` - front button on GPIO41

**Diagnostic**

- `sensor.atom_s3_display_uptime` (if you add an `uptime` sensor block)
- `sensor.atom_s3_display_wifi_signal` (if you add a `wifi_signal` sensor block)

### Entities Consumed from Home Assistant

The display subscribes to and renders these HA entities:

- `time` (from the `homeassistant` time platform)
- `weather.home`
- `sensor.outdoor_temp`
- `light.living_room`, `light.bedroom`, `light.kitchen`
- `sensor.living_room_temp`, `sensor.living_room_humidity`, `sensor.living_room_co2`
- `person.tony`, `person.anna`
- `binary_sensor.front_door`, `binary_sensor.balcony_window`

Rename or remove any of these in the YAML to match your installation.

## Troubleshooting

| Issue | Likely cause | Fix |
|-------|--------------|-----|
| Display stays black after flash | Backlight off, or wrong driver model | Confirm `model: gc9a01a` and that `light.atom_s3_display_display_backlight` is on. Toggle from HA or remove the light block to force always-on. |
| Display shows garbage or shifted pixels | Wrong rotation, or SPI pins differ in a clone module | Try `rotation: 90/180/270`. Confirm CLK=GPIO15, MOSI=GPIO21, CS=GPIO14, DC=GPIO42, RST=GPIO48 - these are the ATOM-S3 defaults. |
| Button does nothing | Pull-up missing or wrong polarity | The YAML uses `pullup: true` and `inverted: true`. Press should drive GPIO41 low. Watch the ESPHome logs while pressing. |
| Page indicator dots animate but text never updates | Home Assistant entities not found | Check `entity_id` spellings. ESPHome logs will print "entity not available" for each missing one. |
| CO2 always shows red | CO2 sensor is over 1000 ppm | That is the intended threshold. Lower it in the lambda if your space stays above 1000 most of the day. |
| OTA fails | WiFi range or password wrong | Re-flash over USB-C. The ATOM-S3 boots into download mode automatically. |
| Time shows 1970 or stays blank | `time.homeassistant` not yet synced | Wait 30-60 seconds after boot; the API has to send the time once. Restart HA's `homeassistant` integration if it stays stuck. |
| Compile fails on font files | TTF files missing from `fonts/` | Create the directory and download Roboto-Regular/Medium/Bold TTFs into it. |
| Display update is slow or stutters | Lambda doing too much work | Reduce `update_interval` work, or move heavy logic into precomputed text sensors. |

## Going Further

Easy upgrades once this is working:

- **Add a buzzer entity** on a spare GPIO and beep when the door opens
- **Long-press detection** on the button to act as a "good night" trigger
- **WiFi signal strength** as a fifth page for diagnostics
- **Animated weather icons** drawn with `display.image` and PNG assets
- **Touch input** by swapping the ATOM-S3 for the ATOMS3 Lite plus an external touch panel
- **Mount with a magnet** on the back of the 3D-printed case for fridge placement
