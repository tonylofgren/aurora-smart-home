# Battery Zigbee Temperature + Humidity Sensor (ESP32-C6)

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

A battery-powered Zigbee end device built on the ESP32-C6. Uses the Zigbee
support that landed in ESPHome core in 2026.5.0, so no external component
or Zigbee gateway code is needed on the device itself. Reports through
standard Zigbee clusters and is picked up automatically by both ZHA and
zigbee2mqtt without quirks or custom converters.

## Features

- **Standard Zigbee clusters** (TEMPERATURE_MEASUREMENT,
  RELATIVE_HUMIDITY_MEASUREMENT) recognized natively by ZHA and z2m
- **Months of battery life** on 2x AA via deep sleep between reports
- **Advertised power source** so the coordinator knows to expect long
  sleeps and won't mark the device offline
- **Stay-awake button** for easy OTA reflashing
- **Battery voltage sensor** so you know when to swap cells
- **Tunable report interval** via a single substitution

## Hardware List

| Component | Purpose | Notes |
|-----------|---------|-------|
| Seeed Studio Xiao ESP32-C6 | Host with Zigbee radio | Cheapest C6 board, USB-C, fits on a stamp-sized PCB |
| SHT40 / SHT45 | Temperature + humidity sensor | I2C, low-power, accurate. SHT31 or Si7021 also work. |
| 2x AA holder + LDO | Power | Or use a 3.7 V LiPo with the dev board's onboard regulator |
| Voltage divider (2x 100 kΩ) | Battery voltage monitor | Optional but recommended |
| 4-pin Dupont cable | I2C wiring | SDA, SCL, 3V3, GND |

You can swap the C6 for an ESP32-H2 dev board if you prefer; the YAML
works on both. H2 has no WiFi, which can extend battery life slightly
since the WiFi stack is never compiled in.

## Installation

### 1. Flash the firmware

USB connection only - no WiFi credentials needed for pure Zigbee devices.

```bash
esphome run zigbee-temp-sensor-c6.yaml --device /dev/ttyACM0
```

On Windows, the device usually appears as `COM3` or higher. Adjust accordingly.

### 2. Wire the sensor

Follow the diagram above. Double-check polarity on the SHT40 (the chip is
sensitive to reverse polarity on VDD).

### 3. Pair with your Zigbee coordinator

**ZHA:**
1. Settings > Devices & Services > ZHA > Add Device
2. Power-cycle the C6 (unplug + replug, or hold reset button)
3. ZHA discovers and adds a "Temperature and Humidity" device

**zigbee2mqtt:**
1. Open "Permit Join" in the Z2M frontend
2. Power-cycle the C6
3. Device joins as a generic temperature/humidity sensor

### 4. Confirm it's reporting

After the first successful join, the device reports immediately and then
sleeps for `sleep_minutes` minutes (default: 15). You'll see new values
arrive in HA every 15 minutes. Battery should last 3-6 months on 2x AA
with a 15-minute interval.

## Configuration

### Adjusting the report interval

Top of the YAML:

```yaml
substitutions:
  sleep_minutes: "15"     # default - good for general use
  # sleep_minutes: "5"    # high-resolution (~1 month battery)
  # sleep_minutes: "60"   # one-per-hour (year-scale battery)
```

### Disabling battery monitoring

If you don't wire the voltage divider, delete the `adc` sensor block. The
firmware boots and works fine without it.

### Adding more clusters

The Zigbee component supports many measurement clusters. To add pressure,
illuminance, occupancy, etc., add another `platform: zigbee` sensor with
the matching `cluster:` key. See `esphome/references/release-2026-5.md`
in the Aurora repo for the full Zigbee section.

## Home Assistant Integration

### Entities Created (via ZHA or zigbee2mqtt, not the ESPHome API)

The device communicates over Zigbee, so it does **not** show up under
the ESPHome HA integration. Look for it under your Zigbee integration
instead.

**Temperature**
- `sensor.zigbee_temp_1_temperature` (or similar, named by your coordinator)

**Humidity**
- `sensor.zigbee_temp_1_humidity`

**Battery**
- `sensor.zigbee_temp_1_battery_voltage` (if you wired the divider)

## Troubleshooting

| Issue | Likely cause | Fix |
|-------|-------------|-----|
| Device never appears in the coordinator | Wrong framework or wrong board variant | Confirm `framework: type: esp-idf` and the C6/H2 board ID. Re-flash. |
| Joined once but values are stuck | Coordinator polled before the SHT40 had a reading | Increase the `delay:` in `on_boot` from 800 ms to 2 s. |
| Battery dies in a week | Logger left at INFO or higher; or sleep_minutes too short | Set `logger: level: WARN`, increase sleep_minutes. |
| Cannot reflash - device sleeps too fast | No way to keep it awake during OTA | Hold the BOOT button (GPIO9) at boot and the device stays awake. |
| Temperature reads ~85 °C | SHT40 power-on default; reading taken too early | Increase the `delay:` after `component.update`. |
| Coordinator reports the device as offline | Sleep interval exceeds the coordinator's keepalive | Lower `sleep_minutes` to 10 or below. Most coordinators tolerate 1 h with sleepy end devices configured correctly. |

## Why ESP32-C6 specifically

| Chip | Zigbee | WiFi | Notes |
|------|:------:|:----:|-------|
| ESP32-C6 | yes | yes | Most flexible. Can be a coexisting WiFi+Zigbee device if you ever need that. |
| ESP32-H2 | yes | no | Even lower power. Best pure-Zigbee battery life. No WiFi means OTA needs USB. |
| ESP32 / S2 / S3 / C3 | no | yes | No Zigbee radio. Not eligible. |

For a "joins a single Zigbee mesh, never touches WiFi" sensor like this
one, the H2 wins on power. The C6 wins on flexibility and is easier to
buy as a dev board at the time of writing.
