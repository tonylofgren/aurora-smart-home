# Smart Plug with Power Monitoring

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

A WiFi smart plug that switches a mains load on or off and reports real-time
voltage, current, power, and accumulated energy back to Home Assistant. It
replicates the firmware behaviour of an Athom Plug V3, Shelly Plug S, or
Sonoff S31, but built on open-source ESPHome so you control the cloud, the
data, and the update channel.

This recipe targets two audiences:

1. **Reflashers.** You bought a cheap Athom or Sonoff plug (around 8 USD)
   and want to put ESPHome on it. This config drops straight in.
2. **DIY builders.** You are putting together an ESP32-C3 + BL0942 module on
   a custom PCB. The hardware is the same; only the pin map changes.

## Safety first

This device sits inline with mains voltage (230 V or 120 V). Mains can kill
you and can start fires. **Do not wire bare ESP boards into mains unless you
are a qualified electrician.** Use a pre-built, certified plug enclosure
(Athom, Shelly, Sonoff) and reflash the firmware instead. Total parts cost
is the same and you get a CE/FCC-rated enclosure with proper isolation.

If you must build your own:

- Use an isolated AC-DC module (HLK-PM03 or similar) with proper creepage.
- Keep mains and low-voltage traces minimum 6 mm apart.
- Use a relay rated for inductive loads if you plan to switch motors.
- House everything in a fire-retardant, vented enclosure.
- Add a fuse on the input.

## Features

- **Remote relay control** via Home Assistant, Google, Alexa, or HomeKit.
- **Power monitoring** through the BL0942 chip (voltage, current, watts).
- **Energy accumulation** with daily totals that feed the HA Energy
  dashboard out of the box.
- **Physical button** for local on/off, with a 5 s hold to clear WiFi.
- **Status LED** that mirrors the relay state.
- **Restore on boot** so the plug remembers if it was on before a power
  glitch.

## Wiring

For the Athom Plug V3 (ESP32-C3) the pin map below is already correct. For
a custom PCB, copy this and rename GPIO numbers in the substitutions block.

```
ESP32-C3
+-- GPIO5  ----- Relay coil driver (active high)
+-- GPIO3  ----- Status LED (active low)
+-- GPIO9  ----- Button to GND (INPUT_PULLUP)
+-- GPIO20 ----- BL0942 TX (becomes RX on ESP)
+-- GPIO21 ----- BL0942 RX (becomes TX on ESP)
+-- 3V3   ----- BL0942 VCC
+-- GND   ----- BL0942 GND, button common
```

## Installation

### 1. Prepare secrets

```bash
cp secrets.yaml.example secrets.yaml
```

Edit `secrets.yaml` and fill in your WiFi credentials, an API encryption
key, and an OTA password. Generate the API key with:

```bash
python -c "import secrets, base64; print(base64.b64encode(secrets.token_bytes(32)).decode())"
```

### 2. First-time flash

Pre-built Athom plug:

1. Open the case, find the 4-pin programming header (3V3, TX, RX, GND).
2. Connect a USB-UART adapter at 3.3 V logic. **Do not plug the device
   into mains while the UART is connected.**
3. Hold the button and apply power to enter boot mode.
4. Run:

   ```bash
   esphome run smart-plug-power-monitor.yaml
   ```

Custom PCB or ESP32-C3 dev board:

```bash
esphome compile smart-plug-power-monitor.yaml
esptool.py --port COM3 write_flash 0x0 .esphome/build/smart-plug/.pioenvs/smart-plug/firmware.bin
```

### 3. Subsequent updates

Once on WiFi, OTA works:

```bash
esphome run smart-plug-power-monitor.yaml --device smart-plug.local
```

### 4. Add to Home Assistant

The device will appear automatically in Settings -> Devices -> ESPHome.
Add the encryption key when prompted.

## Configuration

### Friendly name and multiple plugs

To deploy several plugs from the same YAML, change two substitutions per
unit:

```yaml
substitutions:
  device_name: "washer-plug"
  friendly_name: "Washer Plug"
```

Then flash with a unique hostname:

```bash
esphome run smart-plug-power-monitor.yaml --device washer-plug.local
```

### Reflashing a Sonoff S31

The S31 uses an ESP8266 (ESP8285 actually). Replace the `esp32:` block
with:

```yaml
esp8266:
  board: esp01_1m
  restore_from_flash: true
```

And remap pins:

```yaml
substitutions:
  relay_pin: "GPIO12"
  led_pin: "GPIO13"
  button_pin: "GPIO0"
  bl0942_rx_pin: "GPIO3"
  bl0942_tx_pin: "GPIO1"
```

Note: on the S31 the BL0942 is actually a CSE7766 - swap the sensor
platform from `bl0942` to `cse7766`.

### Calibration

The factory defaults in `substitutions` are accurate to within +-5 % for
Athom V3. To calibrate yourself:

1. Plug in a known resistive load (a 100 W incandescent bulb is ideal).
2. Compare the reported power to a calibrated reference meter.
3. Adjust the `power_ref` substitution proportionally:

   ```
   new_power_ref = old_power_ref * (reported / actual)
   ```

4. Repeat for voltage, current, and energy.

## HA Energy dashboard

`sensor.smart_plug_energy_today` is exposed with
`state_class: total_increasing` and `device_class: energy`. To add it:

1. Settings -> Dashboards -> Energy.
2. Add Consumption -> Individual devices.
3. Pick `sensor.smart_plug_energy_today`.

Daily totals will appear within an hour.

## Automation ideas

The included `automations.yaml` ships with three patterns you can copy:

- **Appliance done** notification when a washer or dishwasher cycle
  finishes (power drops below 2 W for 3 minutes).
- **Standby killer** auto-off after one hour of low standby draw.
- **Overload protection** trips the relay if load exceeds 2300 W.

## Troubleshooting

| Issue                          | Solution                                       |
|--------------------------------|------------------------------------------------|
| Power reads always 0           | UART pins swapped - try flipping rx/tx        |
| Voltage reads ~5 V             | BL0942 not getting 3V3 - check wiring         |
| Relay clicks but no power      | Relay rated too low for the load              |
| Button does nothing            | Wrong GPIO - check `button_pin` substitution  |
| Energy resets randomly         | `restore_from_flash: true` missing on ESP8266 |
| Calibration off by 10 % +      | Re-run calibration with a known load          |

## Compliance

If you sell or distribute a plug like this in the EU or US, you need:

- CE marking (EU): LVD + EMC + RoHS testing
- FCC Part 15 (US): unintentional radiator
- ETL/UL listing (US): nice-to-have for retail

Reflashing a pre-certified Athom or Shelly device does not void its
certification because you are not modifying the hardware.
