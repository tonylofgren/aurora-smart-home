# Multi-Sensor Air Quality Monitor

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

A DIY equivalent of the Awair Element and Apollo MSR-1: an ESP32-S3 based indoor air quality monitor that measures CO2, VOC, NOx, particulate matter, temperature and humidity, with an optional OLED display.

## Features

- Real-time CO2 measurement via NDIR (SCD41), accurate to within 40 ppm
- VOC and NOx index via the Sensirion SGP41 with humidity and temperature compensation
- PM1.0, PM2.5 and PM10 particulate matter via the PMS5003 laser sensor
- Temperature and humidity (SCD41, with configurable self-heating offset)
- Optional 0.96 inch SSD1306 OLED for at-a-glance display of all key values
- Forced outdoor calibration button to keep the SCD41 baseline accurate
- Native Home Assistant API with encryption
- Web server for direct browser access
- Captive portal fallback if Wi-Fi credentials are lost

## Hardware List

| Component | Purpose | Notes |
|-----------|---------|-------|
| ESP32-S3 DevKitC-1 | Main controller | Any ESP32-S3 board works; S3 chosen for memory headroom |
| Sensirion SCD41 | CO2, temperature, humidity | I2C address 0x62, NDIR true CO2 sensor |
| Sensirion SGP41 | VOC index, NOx index | I2C address 0x59, replaces older SGP30 |
| Plantower PMS5003 | PM1.0/PM2.5/PM10 | UART, requires 5V supply for internal fan |
| SSD1306 OLED 128x64 | Local readout (optional) | I2C address 0x3C |
| Project enclosure | Vented housing | Must allow airflow across PMS5003 intake |
| 5V 2A USB-C PSU | Power | PMS5003 fan draws bursts of current |

## Wiring Diagram

```
                    +-------------------+
                    |   ESP32-S3 Dev    |
                    |                   |
        5V  --------+ 5V                |
        GND --------+ GND               |
                    |                   |
                    |  GPIO8 (SDA) +----+----+----+----+
                    |  GPIO9 (SCL) +-+--+--+-+--+-+--+-+
                    |              | |  |  | |  | |  | |
                    |  GPIO17 TX --+ +--+--+ +--+ +--+ |
                    |  GPIO18 RX --+ |  |  | |  | |  | |
                    +--------------+-+--+--+-+--+-+--+-+
                                   | |  |  | |  | |  | |
                       +-----------+ |  |  | |  | |  | |
                       |   PMS5003 | |  |  | |  | |  | |
                       |  (UART)   | |  |  | |  | |  | |
                       +-----------+ |  |  | |  | |  | |
                                     | |  |  | |  | |  | |
                       +-------------+ +--+  | |  | |  | |
                       |   SCD41        |    | |  | |  | |
                       |   0x62 I2C     |    | |  | |  | |
                       +----------------+    | |  | |  | |
                                             | |  | |  | |
                       +---------------------+ +--+ |  | |
                       |   SGP41                    |  | |
                       |   0x59 I2C                 |  | |
                       +----------------------------+  | |
                                                       | |
                       +-------------------------------+ +
                       |   SSD1306 OLED (optional)
                       |   0x3C I2C
                       +-------------------------------
```

All I2C devices share the same SDA/SCL pair (GPIO8/GPIO9). The PMS5003 uses a dedicated UART on GPIO17/GPIO18 at 9600 baud. PMS5003 Vcc must be 5V; logic lines are already 3.3V.

## Calibration

The SCD41 ships with a factory CO2 baseline that drifts a few ppm per week. To keep it accurate:

1. Once per week, place the unit outdoors in clean air, away from cars and people.
2. Wait at least 3 minutes for the sensor to stabilise.
3. In Home Assistant, press the `Air Quality force SCD41 outdoor calibration` button.
4. The device writes a 420 ppm baseline (a typical clean outdoor value) to the SCD41 EEPROM.

Automatic self-calibration (ASC) is also enabled. ASC assumes the unit sees an outdoor-fresh-air-level CO2 reading at least once per week, which is true in most homes that get ventilated.

The PMS5003 and SGP41 are not user-calibratable. The VOC/NOx index is relative: 100 is the running 24 hour average, higher values mean worse than baseline. Give the SGP41 about 12 hours of runtime to learn its environment.

## Installation

1. Copy `secrets.yaml.example` to `secrets.yaml` and fill in your Wi-Fi credentials, API key and OTA password.
2. Wire the sensors per the diagram above.
3. Install ESPHome and run `esphome run air-quality-multi.yaml` for the initial USB flash.
4. After the device boots and joins Wi-Fi, all future updates can be done OTA from the Home Assistant ESPHome add-on or the ESPHome CLI.
5. In Home Assistant, the device is auto-discovered via mDNS. Add it from Settings -> Devices & Services -> ESPHome.
6. Copy the contents of `automations.yaml` into your `configuration.yaml` or a packages file.
7. Add the cards from `dashboard.yaml` to your Lovelace dashboard.

## Configuration

The YAML uses substitutions at the top of the file. Change these to rename the device:

```yaml
substitutions:
  devicename: "air-quality"
  long_devicename: "Air Quality"
```

To disable the optional OLED, comment out or delete the entire `font:` and `display:` blocks. The firmware will still build and all sensors continue to publish to Home Assistant.

To change the sensor update interval, edit `update_interval: 60s` on each sensor. 60 seconds is recommended; faster sampling wastes power on the PMS5003 fan and produces no useful resolution because air composition changes slowly.

## Air Quality Reference

**CO2 (ppm)**

| Range | Status | Effect |
|-------|--------|--------|
| < 800 | Good | Outdoor-equivalent, no measurable cognitive impact |
| 800 to 1200 | OK | Acceptable for living spaces, mild stuffiness possible |
| 1200 to 2000 | Poor | Drowsiness, reduced focus, ventilate now |
| > 2000 | Bad | Headache risk, open windows or leave the room |

**PM2.5 (ug/m3, WHO 2021 guideline)**

| Range | Status |
|-------|--------|
| 0 to 12 | Good |
| 12 to 35 | Moderate |
| 35 to 55 | Unhealthy for sensitive groups |
| 55 to 150 | Unhealthy |
| > 150 | Very unhealthy |

**VOC index (Sensirion scale)**

| Range | Meaning |
|-------|---------|
| 0 to 100 | Below 24 hour baseline (cleaner than usual) |
| 100 | The running 24 hour baseline for this room |
| 100 to 250 | Above baseline, mild source present (cooking, candles, cleaning) |
| 250 to 400 | Strong source (solvent, off-gassing, fresh paint) |
| > 400 | Very strong source, investigate |

**NOx index** uses the same scale but with a baseline of 1. Values above 20 indicate combustion or NO2 ingress.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---------|--------------|-----|
| CO2 reads `unknown` for the first minute | SCD41 first measurement cycle | Normal, resolves after 60 s |
| CO2 always reads exactly 400 ppm | Sensor not detected on I2C | Check 0x62 in `i2c.scan` log, verify wiring |
| Temperature reads 3-5 C high | Self-heating inside enclosure | Adjust `temperature_offset` in YAML |
| PM2.5 stuck at 0 | PMS5003 fan not spinning | Verify 5V on Vcc, fan should be audible |
| VOC index stuck at 100 | Sensor still learning | Give it 12 hours uninterrupted runtime |
| OLED shows nothing but sensors work | I2C address mismatch | Some clones use 0x3D, edit `address:` line |
| Device drops Wi-Fi at night | Power supply sag during PMS5003 burst | Use a 2A or larger 5V supply |
| Forced calibration button does nothing | Unit was indoors when pressed | Run outdoors for 3 minutes first |
