# Panasonic Heat Pump Example

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

Local-only Panasonic AC control over the CN-CNT or CN-WLAN service connector. Reports real temperature, full mode/fan/swing control, outside temperature, estimated power consumption, and the brand-specific features (Econavi, NanoeX, mild dry, eco mode).

## Features

- **Full climate entity** in Home Assistant: mode, target temperature, fan speed, swing
- **Outside temperature sensor** read directly from the AC main board
- **Estimated power consumption sensor**
- **Econavi + NanoeX + mild dry + eco mode** as separate HA switches (per-AC support)
- **Vertical and horizontal swing select** entities
- **Status LED** and **restart switch** for diagnostics
- **No cloud dependency** - direct UART to the AC, no Comfort Cloud app needed

## Hardware List

### ESP32 Controller

| Component | Purpose | Notes |
|-----------|---------|-------|
| ESP32 dev board | Host | ESP32-WROOM-32 or any equivalent. ESP8266 not recommended (only one hardware UART). |

### Level shifter (mandatory)

| Component | Purpose |
|-----------|---------|
| TXS0108E or BSS138-based 4-channel module | 5 V <-> 3.3 V bi-directional translation between ESP UART and AC service port |

### AC service connector

| Component | Purpose |
|-----------|---------|
| 5-pin Molex CN05 plug (CN-CNT) or 4-pin (CN-WLAN) | Connects to the AC main board service port. Hand-crimp pins or buy a pre-made cable. |
| Female-female jumper wires | ESP <-> level shifter <-> AC connector |

## Safety

> **The AC main board operates on 5 V logic. ESP32 GPIOs are 3.3 V.**
> Connecting them directly **will damage the AC main board.** Replacement boards typically cost more than the AC itself.

Always wire through a bi-directional logic level converter. The TXS0108E is widely available, costs around 2 USD, and is what every other ESPHome-on-Panasonic build uses. Do not skip this.

## AC compatibility

This example targets Panasonic split AC units with one of these service connectors:

| Port | Module name | `type:` value | Era |
|------|-------------|---------------|-----|
| CN-CNT | CZ-TACG1 | `type: cnt` (default) | Older units |
| CN-WLAN | DNSK-P11 | `type: wlan` | Newer units (can coexist with CN-CNT) |

Check the inside of the AC's plastic cover for the connector labels before buying parts. Some newer units have only CN-WLAN, some older units have only CN-CNT, and a few have both.

Per-type feature differences:
- `eco_switch` and `mild_dry_switch` work on `type: cnt` only.
- All other entities work on both.

## Wiring Diagram

```
ESP32 dev board                  TXS0108E level shifter              Panasonic CN-CNT
                                 (or BSS138 4-channel)
+----------------+               +--------------------+              +--------------+
|         3V3 ---|--- VccA ----- |                    | --- VccB ----|--- +5V (pin 5)
|         GND ---|--- GND ------ |                    | --- GND -----|--- GND (pin 4)
|       GPIO17 --|--- A1 (3.3V)- | --- B1 (5V) -------|--- TX  (pin 2)
|       GPIO16 --|--- A2 (3.3V)- | --- B2 (5V) -------|--- RX  (pin 1)
|         GPIO2 -|--- (status LED on most dev boards)
+----------------+               +--------------------+              +--------------+

Pin 3 on the CN-CNT connector is not used. Some Panasonic models use a
different pinout; verify against the DomiStyle hardware guide for your
exact model before powering on.
```

A 940 nm IR LED + transistor driver is **not** used in this example. The wired UART path replaces it entirely.

## Installation

### 1. Vendor the panasonic_ac component into your project

From the Aurora repo, copy the entire `esphome/components/panasonic_ac/` directory next to your project YAML so the layout becomes:

```
your-project/
├── panasonic-heatpump.yaml
└── components/
    └── panasonic_ac/
        ├── LICENSE
        ├── NOTICE.md
        ├── __init__.py
        ├── climate.py
        ├── esppac.h
        ├── esppac.cpp
        └── ... (10 more files)
```

The `LICENSE` and `NOTICE.md` files must travel with the code. They are MIT and required by the upstream license.

### 2. Alternative: load from upstream GitHub

If you do not want a local copy, edit `panasonic-heatpump.yaml` to comment out the local `external_components:` block and uncomment the GitHub one:

```yaml
external_components:
  - source: github://DomiStyle/esphome-panasonic-ac
    components: [panasonic_ac]
```

This fetches the same code from upstream at compile time. Needs internet during the build.

### 3. Configure secrets

```bash
cp secrets.yaml.example secrets.yaml
# edit secrets.yaml and fill in wifi_ssid, wifi_password, etc.
```

### 4. Wire the hardware

1. Open the AC's plastic cover (look for screws on the underside or side panels).
2. Identify the CN-CNT or CN-WLAN connector on the main board.
3. Crimp or solder a cable to a 5-pin (CN-CNT) or 4-pin (CN-WLAN) plug.
4. Wire ESP <-> level shifter <-> AC connector per the diagram above.
5. **Double-check polarity and voltages before plugging in.** The 5 V from the AC powers the high side of the level shifter; the ESP supplies its own 3.3 V to the low side.

### 5. Flash the ESP32

```bash
esphome run panasonic-heatpump.yaml
```

### 6. Plug into the AC and verify

After flashing and connecting:
- Home Assistant should discover the new device via the ESPHome integration.
- The climate entity should show current temperature within a minute.
- Open the ESPHome dashboard logs and confirm packets are flowing both directions on the UART.

### 7. Optional: add automations + dashboard

Copy the contents of `automations.yaml` and `dashboard.yaml` to your Home Assistant configuration.

## Configuration

### Switching between CNT and WLAN

Top of the YAML:

```yaml
substitutions:
  ac_type: "cnt"      # change to "wlan" for DNSK-P11 / CN-WLAN units
```

### Enabling NanoeX

Uncomment the `nanoex_switch:` block in the `climate:` section. Most Panasonic mid-range units from 2018 onwards have NanoeX; check your physical remote for the button to confirm.

### Eco mode and mild dry (CNT only)

If you have a `type: cnt` AC and the physical remote shows Eco / Mild Dry buttons, uncomment the corresponding switch blocks. Do **not** enable these on a `type: wlan` unit - the WLAN protocol does not implement them and the AC behavior is undefined.

### Temperature sensor offsets

If the AC reports a temperature that disagrees with a reference thermometer, add an offset:

```yaml
climate:
  - platform: panasonic_ac
    current_temperature_offset: -0.5    # subtract 0.5 °C from reported value
    outside_temperature_offset: 0
```

## Home Assistant Integration

### Entities Created

**Climate**
- `climate.panasonic_ac` - main entity (mode, target temp, fan speed, current temp)

**Selects**
- `select.panasonic_ac_vertical_swing` - up / up_center / center / down_center / down / auto / swing
- `select.panasonic_ac_horizontal_swing` - left / left_center / center / right_center / right / auto

**Sensors**
- `sensor.panasonic_ac_outside_temperature`
- `sensor.panasonic_ac_estimated_power`
- `sensor.panasonic_ac_uptime`

**Switches**
- `switch.panasonic_ac_econavi`
- `switch.panasonic_ac_restart`
- *(optional, uncomment in YAML)* `switch.panasonic_ac_nanoex`
- *(CNT only, uncomment in YAML)* `switch.panasonic_ac_eco_mode`, `switch.panasonic_ac_mild_dry`

**Light**
- `light.panasonic_ac_status_led` - the ESP's onboard LED. Turn off if it bothers you.

## Troubleshooting

| Issue | Likely cause | Fix |
|-------|-------------|-----|
| Climate entity never shows current temperature | UART not connected or polarity reversed | Swap TX/RX on the ESP side. Verify level shifter is powered on both sides (3.3 V and 5 V LEDs lit). |
| Compile fails on `panasonic_ac` not found | Local component directory missing or wrong path | Confirm `components/panasonic_ac/` sits next to the YAML and contains `__init__.py` + `climate.py`. |
| Schema rejects `eco_switch` or `mild_dry_switch` | You set `type: wlan` but enabled CNT-only entities | Comment out those switches, they are not supported on DNSK-P11. |
| AC behaves weirdly after enabling NanoeX or Econavi | Your unit does not actually have that feature | Disable the entity, check the physical remote for the corresponding button. |
| Watchdog reset every few minutes | UART parity wrong, framing errors flood the loop | Confirm `parity: EVEN`. Some clones of the DNSK-P11 use 8N1 instead - try `parity: NONE` only if EVEN does not work. |
| HA shows ECO on but the AC light is off | State drift from physical remote use | Force a sync by toggling the entity in HA, or wait for the next poll cycle. |
