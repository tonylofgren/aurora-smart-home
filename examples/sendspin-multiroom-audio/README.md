# Sendspin Multi-Room Synchronized Audio

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

Whole-house synchronized audio using ESPHome 2026.5.0's new Sendspin
component family. One small device acts as the hub (and exposes the
group player to Home Assistant); every other device plays the same
audio in lock-step over your existing LAN. Real-time stereo Opus
playback works on a plain ESP32 in this release.

## Features

- **One group media_player in HA** that drives every speaker at once
- **Per-room delay compensation** to align rooms with different
  DAC/amp latency
- **Track metadata sensors** (title, artist, album, progress) for
  driving wall-panel displays
- **Scales to as many rooms as your WiFi can carry** - 4-6 speakers is
  typical, more works if your network is healthy
- **Built entirely from ESP32 hardware** - no proprietary speakers, no
  cloud, no subscription

## Hardware List

### Hub device (one per house)

| Component | Purpose | Notes |
|-----------|---------|-------|
| ESP32-S3 dev board with PSRAM | Hub | PSRAM gives the Opus decoder room to breathe. Plain ESP32 also works. |
| USB-C power supply | Power | Hub is always on; should be near the router |

The hub doesn't need an attached speaker - it can be headless and just
control the group. If you want it to also play audio in its own room,
add the same I2S amp + speaker hardware as the per-room speakers.

### Per-room speaker device (one per room)

| Component | Purpose | Notes |
|-----------|---------|-------|
| ESP32 dev board (any variant) | Host | Plain ESP32-WROOM is fine since 2026.5.0 |
| MAX98357A I2S amplifier | Audio output | ~3 USD, very common. PCM5102A is a higher-fi alternative. |
| 4 ohm 3 W speaker | Acoustic output | Match the amp. Larger speakers need a larger amp. |
| USB-C or barrel-jack 5 V supply | Power | Speakers can be far from the router; just need WiFi reach |

## Wiring Diagram (per speaker)

```
ESP32 dev board                 MAX98357A I2S amp           Speaker
+----------------+              +----------------+
|         5V ----|--- VIN ----- |                |
|         GND ---|--- GND ----- |                |          +-----+
|       GPIO22 --|--- DIN ----- |        OUT+ ---|----------|+    |
|       GPIO25 --|--- LRC ----- |        OUT- ---|----------|-    |
|       GPIO26 --|--- BCLK ---- |                |          +-----+
|                |              |   (SD floating |
|                |              |    for mono)   |
+----------------+              +----------------+
```

The `SD` pin on the MAX98357A selects mono channel (left, right, or
left+right summed). Leave it floating for left+right summed - simplest
for single-speaker rooms.

## Installation

### 1. Plan the network

- Pick a static IP for the hub (the example uses `192.168.1.50`).
  Reserve it in your DHCP server or configure your router to always
  hand the hub the same address.
- All speakers must be on the same broadcast domain as the hub. Sendspin
  uses multicast UDP for sync.

### 2. Flash the hub

```bash
esphome run audio-hub.yaml
```

Verify it boots, joins WiFi, and gets the static IP you reserved. Open
its web UI at `http://192.168.1.50/`.

### 3. Flash each room speaker

For each room, copy `speaker.yaml` to a new file (e.g.
`speaker-bedroom.yaml`) and change two lines:

```yaml
substitutions:
  devicename: "speaker-bedroom"      # unique per device
  room_name: "Bedroom"               # human-readable label
```

Then:

```bash
esphome run speaker-bedroom.yaml
```

Repeat for kitchen, hallway, etc.

### 4. Verify in Home Assistant

After every speaker is online, you should see:

- `media_player.audio_hub_all_rooms` - the group control
- `media_player.living_room_speaker`, `media_player.bedroom_speaker`,
  etc. - one per room

Press play on the group entity. You should hear synchronized audio in
every room within a second.

### 5. Tune delay_compensation

If one room sounds slightly ahead of the others (audible doubling),
edit that room's YAML and raise `delay_ms` by 10 ms. Re-flash. Repeat
until the doubling is gone.

Different DAC/amp chains have different latency. Typical values:

| Output chain | Typical delay_compensation |
|--------------|----------------------------|
| MAX98357A direct | 40-60 ms |
| PCM5102A + small amp | 50-80 ms |
| Through an AV receiver's analog input | 100-150 ms |
| Through a digital amp (DSP latency) | 100-200 ms |

## Configuration

### Adjusting the report interval / static IP

`audio-hub.yaml`:

```yaml
substitutions:
  hub_ip: "192.168.1.50"     # change to match your DHCP reservation
  hub_port: "7474"           # default; only change if 7474 is taken
```

Speaker YAML files **must** reference the same hub_ip and hub_port:

```yaml
substitutions:
  hub_ip: "192.168.1.50"
  hub_port: "7474"
```

### Mono vs stereo per room

In `speaker.yaml`:

```yaml
speaker:
  - platform: i2s_audio
    channel: mono              # or 'stereo' if you have two amps + two speakers
```

True stereo needs two MAX98357A modules (one for left, one for right)
with the SD pin tied appropriately.

## Home Assistant Integration

### Entities Created

**Group (on the hub)**
- `media_player.audio_hub_all_rooms` - play this for whole-house audio

**Per room (one set per speaker)**
- `media_player.<room>_speaker` - individual room control

**Metadata (on the hub)**
- `sensor.audio_hub_progress` - 0 to 100, drives a progress bar
- `text_sensor.audio_hub_now_playing_title`
- `text_sensor.audio_hub_now_playing_artist`
- `text_sensor.audio_hub_now_playing_album`

**Diagnostics**
- `sensor.audio_hub_uptime` and per-speaker uptimes
- `switch.audio_hub_restart` and per-speaker restart switches

## Troubleshooting

| Issue | Likely cause | Fix |
|-------|-------------|-----|
| One room is silent | WiFi signal weak in that room | Move the speaker, add a WiFi repeater, or wire it on Ethernet (ESP32 supports it with an external module) |
| Rooms drift after a few minutes | Hub static IP changed | Make sure the hub has a real DHCP reservation. |
| Audible doubling in one room | delay_compensation off | Raise the value for that room in 10 ms steps. |
| Hub never appears in HA | API key mismatch or WiFi not joined | Open the hub web UI directly to verify it's online. Check the API key matches across hub and speakers if using cross-device features. |
| Stuttering at high volume | DAC clipping | Reduce HA volume; the I2S signal also clips if you set ESPHome volume above ~80% on small amps. |
| Speakers join then drop offline | Multicast disabled on your router/AP | Check that IGMP snooping is enabled on your switch / AP, OR turn it OFF if it's misconfigured. |

## How sync actually works

Sendspin uses multicast UDP for audio distribution. The hub timestamps
each audio packet with an NTP-derived wall-clock value. Each speaker
buffers a small amount of audio (controlled by `delay_compensation`)
and plays the audio at the timestamp written by the hub, regardless of
when the packet actually arrived.

Network jitter is absorbed by the buffer, so as long as a packet arrives
within `delay_compensation` of its scheduled play time, sync holds. If
a packet arrives later than that, the speaker briefly drops to silence
and re-syncs with the next packet. You hear that as a soft dropout.

## Why this matters

Before 2026.5.0, multi-room audio on ESPHome required vendor-specific
hardware (Sonos clones, AirPlay shims) or custom Python on a Pi running
Snapcast. Sendspin is the first first-class implementation that ships
with ESPHome itself, runs on $5 hardware per room, and integrates
directly with Home Assistant as a normal media_player.
