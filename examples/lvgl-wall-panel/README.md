# LVGL Wall Panel

> *Generated by [aurora@aurora-smart-home (esphome skill)](https://github.com/tonylofgren/aurora-smart-home)*

Wall-mounted Home Assistant control panel built on a sub-$20 Sunton touchscreen module and ESPHome 2026.5.0's improved LVGL stack. Three pages of native-feeling controls for lights, climate, and multi-room audio, with PWM backlight dimming and automatic sleep after 30 s of no touch.

## Features

- **Three swipeable pages**: lights, climate, music
- **Time + weather row** at the top of the lights page
- **Four light brightness sliders** for living room, bedroom, kitchen, hallway (driving HA `number.*_brightness` entities)
- **Thermostat page** with large setpoint readout and up/down buttons
- **Music page** with play/pause/next/prev and a group volume slider for the Sendspin multi-room audio hub
- **PWM backlight** dimmed automatically after 30 s idle, restored on first touch
- **Uses 2026.5.0 LVGL improvements**: `grid_rows: "3x"` shorthand, `on_pressed` lambda with `point.x` / `point.y`, `on_update` trigger to separate user vs programmatic value changes
- **Local-only** - no cloud, no proprietary firmware, runs on a $15 board

## Hardware List

### Reference board (ESP32 + 2.8" screen)

| Component | Purpose | Notes |
|-----------|---------|-------|
| Sunton ESP32-2432S028R | All-in-one: ESP32-WROOM-32, 2.8" 240x320 ILI9341 display, XPT2046 resistive touch, backlight PWM, SD slot | About 15 USD on AliExpress; sold as "ESP32 Cheap Yellow Display" (CYD) |
| 5 V USB-C supply | Power | The board has a CH340 USB-C input |
| Optional wall mount / 3D-printed bezel | Fit-and-finish | Many free models on Printables for the 2432S028R |

### Alternative board (ESP32-S3 + 4.3" screen)

| Component | Purpose | Notes |
|-----------|---------|-------|
| Sunton ESP32-S3-4827S043 | ESP32-S3 with PSRAM, 4.3" 480x272 ILI9488, capacitive touch | About 35 USD; capacitive touch is noticeably nicer than resistive |

For the 4.3" board, change `model: ILI9341` to `model: ILI9488`, swap the `touchscreen` platform to the capacitive one your panel ships with, and use the alternate `esp32:` block included at the top of the YAML.

## Wiring

The Sunton boards are pre-wired internally. The YAML uses the documented pinout for the ESP32-2432S028R:

```
Function           ESP32 GPIO
Display SPI CLK    GPIO14
Display SPI MOSI   GPIO13
Display SPI MISO   GPIO12
Display CS         GPIO15
Display DC         GPIO2
Display RESET      GPIO4
Backlight PWM      GPIO21
Touch SPI CLK      GPIO25
Touch SPI MOSI     GPIO32
Touch SPI MISO     GPIO39
Touch CS           GPIO33
Touch IRQ          GPIO36
```

If you wire your own ILI9341 + XPT2046 panel to a generic ESP32 dev board, match these pins or change them in the YAML to suit.

## Installation

### 1. Pick the board

If you bought a Sunton ESP32-2432S028R, you are ready. Otherwise edit the `esp32:` block and the `display:` / `touchscreen:` pins at the top of `lvgl-wall-panel.yaml` to match your hardware.

### 2. Configure secrets

```bash
cp secrets.yaml.example secrets.yaml
# edit secrets.yaml and fill in wifi_ssid, wifi_password, etc.
```

### 3. Create the Home Assistant entities the panel drives

The UI assumes these entities already exist in Home Assistant. Add them in `configuration.yaml` (or via the UI):

```yaml
input_number:
  thermostat_setpoint:
    name: Thermostat setpoint
    min: 16
    max: 28
    step: 1
    unit_of_measurement: "C"
  audio_group_volume:
    name: Audio group volume
    min: 0
    max: 100
    step: 1

  living_room_light_brightness:
    name: Living room brightness
    min: 0
    max: 100
  bedroom_light_brightness:
    name: Bedroom brightness
    min: 0
    max: 100
  kitchen_light_brightness:
    name: Kitchen brightness
    min: 0
    max: 100
  hallway_light_brightness:
    name: Hallway brightness
    min: 0
    max: 100
```

Expose them to ESPHome as `number.*` entities through the standard `input_number` -> `number` mapping. If you already drive your lights from an HA dashboard, point the sliders at your existing `number.*` or `light.*` entities by editing the `homeassistant.action:` block of each slider.

### 4. Flash

```bash
esphome run lvgl-wall-panel.yaml
```

First flash needs USB. After that, all updates go over the air via the ESPHome dashboard.

### 5. Verify

After the panel boots and joins WiFi:

- Home Assistant should discover the new device via the ESPHome integration.
- The lights page should show the four sliders, with their initial position reflecting the linked `number.*_brightness` entity.
- Touching the screen should bring the backlight to 80 % and reset the 30 s idle timer.
- Leave the panel alone for 30 s and the backlight should fade to off; touching it should bring it back instantly.

### 6. Optional: add automations + dashboard

Copy the contents of `automations.yaml` and `dashboard.yaml` to your Home Assistant configuration.

## Configuration

### Adjusting the idle timeout

Top of the YAML:

```yaml
substitutions:
  idle_timeout: "30s"   # raise to "2min" if you want it to stay lit longer
```

The same value is used by the `script:` that wakes the backlight on touch and by the `lvgl: on_idle:` trigger that fades it back out.

### Recalibrating touch

If touches land in the wrong place, dump raw values from the XPT2046 by raising the logger level to DEBUG and watching the serial / web log while you press the four corners. Update the `touchscreen: calibration:` block with the observed min/max values, then re-flash.

### Theming

The `lvgl: theme:` block at the top of the `lvgl:` section sets default colors for every button and slider. Change `bg_color` and `text_color` to retheme the entire UI in one place. Hex values are RGB888.

## Troubleshooting

| Issue | Likely cause | Fix |
|-------|-------------|-----|
| Screen stays black after boot | Wrong display model or backlight wiring | Confirm `model: ILI9341` for the 2.8" board; raise `light.turn_on` brightness to 100 % to rule out PWM at 0. |
| Image is mirrored or sideways | `touchscreen.transform` or `display.rotation` wrong | Toggle `mirror_x` / `mirror_y` / `swap_xy` in the YAML and re-flash. |
| Touch is offset from the visible cursor | XPT2046 calibration is way off | Replace the `calibration:` block with values logged from your own panel (see Configuration section). |
| Backlight never dims | `on_idle` not firing | Confirm the `lvgl: on_idle: timeout:` and the `script.reset_idle_timer` both use the same `${idle_timeout}` value. |
| Sliders snap back to old value after release | HA `number.*` entity has a different range or step than the slider | Change `min_value` / `max_value` on the slider to match the HA entity. |
| LVGL crashes on boot with "out of memory" | PSRAM disabled or framebuffer too large | Confirm `psram:` block is present and the board actually has PSRAM (the 2432S028R does not; the S3 4.3" board does). |
| Compile warns "ignore_strapping_warning" on RESET | GPIO4 is a strapping pin on ESP32 | Already handled in the YAML by `ignore_strapping_warning: true`; leave it on. |

## Why this example

LVGL on ESPHome stabilised over the last few releases, and 2026.5.0 added the small quality-of-life features that make a real wall panel maintainable: grid shorthand cuts the YAML in half, `on_pressed` with `point.x/point.y` lets you build gesture-like behavior, and `on_update` finally distinguishes user-driven value changes from programmatic ones, which removes a class of feedback loops that used to plague slider widgets bound to Home Assistant entities.

Combine that with the Sunton ESP32 boards which arrive pre-wired for $15 and you get a credible alternative to commercial wall panels (Inelo, Mirabella, etc.) that costs roughly a tenth as much and never phones home.
