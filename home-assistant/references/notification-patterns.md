# Notification Patterns

Guide for implementing smart notifications in Home Assistant.

## Overview

Good notifications are:
- **Relevant** - Sent only when needed
- **Actionable** - Contain actions the user can take
- **Context-aware** - Adapted to time, location, and situation
- **Non-intrusive** - Respect do-not-disturb modes

## Notification Platforms

| Platform | Use case | Actions | Images |
|----------|----------|---------|--------|
| Mobile App | Primary for people | ✅ Yes | ✅ Yes |
| Persistent | Dashboard messages | ❌ No | ❌ No |
| TTS | Voice announcements | ❌ No | ❌ No |
| Email | Reports, logs | ❌ No | ✅ Yes |
| Telegram | Chat-based | ✅ Yes | ✅ Yes |
| Slack/Discord | Team notifications | ✅ Yes | ✅ Yes |

## Mobile App Notifications

### Basic Notification

```yaml
# Generated by aurora@aurora-smart-home (home-assistant skill) v1.7.9
# https://github.com/tonylofgren/aurora-smart-home

automation:
  - alias: "Notification - Simple"
    id: notification_simple
    triggers:
      - trigger: state
        entity_id: binary_sensor.front_door
        to: "on"
    actions:
      - action: notify.mobile_app_tonys_iphone
        data:
          title: "Door opened"
          message: "The front door was just opened"
```

### With Action Buttons

```yaml
automation:
  - alias: "Notification - With actions"
    id: notification_with_actions
    triggers:
      - trigger: state
        entity_id: binary_sensor.motion_garage
        to: "on"
    conditions:
      - condition: state
        entity_id: person.tony
        state: "not_home"
    actions:
      - action: notify.mobile_app_tonys_iphone
        data:
          title: "Motion in the garage"
          message: "Motion detected while you are away"
          data:
            actions:
              - action: "VIEW_CAMERA"
                title: "View camera"
                icon: "sfsymbols:camera"
              - action: "ALARM_ON"
                title: "Arm alarm"
                icon: "sfsymbols:lock.shield"
                destructive: true
              - action: "IGNORE"
                title: "Ignore"
```

### Handling Actions

```yaml
automation:
  - alias: "Handle - View camera"
    id: handle_view_camera
    triggers:
      - trigger: event
        event_type: mobile_app_notification_action
        event_data:
          action: "VIEW_CAMERA"
    actions:
      - action: notify.mobile_app_tonys_iphone
        data:
          title: "Garage Camera"
          message: "Current snapshot"
          data:
            image: "/api/camera_proxy/camera.garage"

  - alias: "Handle - Arm alarm"
    id: handle_alarm_on
    triggers:
      - trigger: event
        event_type: mobile_app_notification_action
        event_data:
          action: "ALARM_ON"
    actions:
      - action: alarm_control_panel.alarm_arm_away
        target:
          entity_id: alarm_control_panel.home
```

## Context-Aware Notifications

### Based on Home Status

```yaml
automation:
  - alias: "Smart notification - Home status"
    id: smart_notification_home_status
    triggers:
      - trigger: state
        entity_id: binary_sensor.motion_backyard
        to: "on"
    actions:
      - choose:
          # Home - No message
          - conditions:
              - condition: state
                entity_id: binary_sensor.anyone_home
                state: "on"
            sequence: []

          # Away - Notification with actions
          - conditions:
              - condition: state
                entity_id: binary_sensor.anyone_home
                state: "off"
            sequence:
              - action: notify.mobile_app_tonys_iphone
                data:
                  title: "Motion outdoors"
                  message: "Motion in the backyard while you are away"
                  data:
                    actions:
                      - action: "VIEW_CAMERA"
                        title: "View camera"
```

### Based on Time

```yaml
automation:
  - alias: "Smart notification - Time"
    id: smart_notification_time
    triggers:
      - trigger: state
        entity_id: lock.front_door
        to: "unlocked"
    actions:
      - choose:
          # Night - Critical
          - conditions:
              - condition: time
                after: "23:00:00"
                before: "06:00:00"
            sequence:
              - action: notify.mobile_app_tonys_iphone
                data:
                  title: "⚠️ Door unlocked"
                  message: "The front door was unlocked in the middle of the night!"
                  data:
                    push:
                      sound:
                        critical: 1

          # Day - Normal
          - conditions:
              - condition: time
                after: "06:00:00"
                before: "23:00:00"
            sequence:
              - action: notify.mobile_app_tonys_iphone
                data:
                  title: "Door unlocked"
                  message: "The front door was unlocked"
```

### Based on Focus Mode (iPhone)

```yaml
automation:
  - alias: "Respect focus mode"
    id: respect_focus_mode
    triggers:
      - trigger: state
        entity_id: sensor.motion_living_room
        to: "on"
    conditions:
      # Only send if not in Do Not Disturb
      - condition: not
        conditions:
          - condition: state
            entity_id: sensor.tonys_iphone_focus_mode
            state: "Do Not Disturb"
    actions:
      - action: notify.mobile_app_tonys_iphone
        data:
          message: "Motion in the living room"
```

## Notification Groups

Send to multiple devices:

```yaml
# configuration.yaml
notify:
  - platform: group
    name: all_phones
    services:
      - action: mobile_app_tonys_iphone
      - action: mobile_app_annas_iphone

  - platform: group
    name: adults
    services:
      - action: mobile_app_tonys_iphone
      - action: mobile_app_annas_iphone

  - platform: group
    name: parents
    services:
      - action: mobile_app_tonys_iphone
      - action: mobile_app_annas_iphone
```

```yaml
automation:
  - alias: "Notify everyone"
    triggers:
      # ...
    actions:
      - action: notify.all_phones
        data:
          title: "Message to everyone"
          message: "This goes to all phones"
```

## Persistent Notifications

Dashboard messages:

```yaml
automation:
  - alias: "Persistent - Update available"
    id: persistent_update_available
    triggers:
      - trigger: state
        entity_id: update.home_assistant_core_update
        to: "on"
    actions:
      - action: persistent_notification.create
        data:
          title: "Update available"
          message: >
            Home Assistant {{ state_attr('update.home_assistant_core_update', 'latest_version') }}
            is available. You are running {{ state_attr('update.home_assistant_core_update', 'installed_version') }}.
          notification_id: "ha_update"

  # Remove once updated
  - alias: "Persistent - Remove update notice"
    id: persistent_remove_update
    triggers:
      - trigger: state
        entity_id: update.home_assistant_core_update
        to: "off"
    actions:
      - action: persistent_notification.dismiss
        data:
          notification_id: "ha_update"
```

## Summary Notifications

Daily summary:

```yaml
automation:
  - alias: "Daily summary"
    id: daily_summary
    triggers:
      - trigger: time
        at: "08:00:00"
    conditions:
      - condition: state
        entity_id: binary_sensor.workday_sensor
        state: "on"
    actions:
      - action: notify.mobile_app_tonys_iphone
        data:
          title: "☀️ Good morning"
          message: >
            Weather: {{ states('weather.home') }}, {{ state_attr('weather.home', 'temperature') }}°C

            {% set lights = states.light | selectattr('state', 'eq', 'on') | list | count %}
            💡 {{ lights }} lights on

            {% set open_windows = states.binary_sensor
              | selectattr('attributes.device_class', 'eq', 'window')
              | selectattr('state', 'eq', 'on')
              | list | count %}
            {% if open_windows > 0 %}
            🪟 {{ open_windows }} windows open
            {% endif %}

            {% set battery_low = states.sensor
              | selectattr('attributes.device_class', 'eq', 'battery')
              | selectattr('state', 'lt', '20')
              | list %}
            {% if battery_low | count > 0 %}
            🔋 Low battery: {{ battery_low | map(attribute='name') | join(', ') }}
            {% endif %}
```

## Rate Limiting

Avoid notification spam:

```yaml
automation:
  - alias: "Rate limited notification"
    id: rate_limited_notification
    triggers:
      - trigger: state
        entity_id: binary_sensor.motion
        to: "on"
    conditions:
      # Max one notification per 5 minutes
      - condition: template
        value_template: >
          {{ (as_timestamp(now()) - as_timestamp(
              state_attr('automation.rate_limited_notification', 'last_triggered') or 0
          )) > 300 }}
    actions:
      - action: notify.mobile_app_tonys_iphone
        data:
          message: "Motion detected"
```

Or with input_datetime:

```yaml
# Helper
input_datetime:
  last_motion_notification:
    has_date: true
    has_time: true

automation:
  - alias: "Rate limited with helper"
    id: rate_limited_helper
    triggers:
      - trigger: state
        entity_id: binary_sensor.motion
        to: "on"
    conditions:
      - condition: template
        value_template: >
          {{ (now() - states('input_datetime.last_motion_notification') | as_datetime).total_seconds() > 300 }}
    actions:
      - action: input_datetime.set_datetime
        target:
          entity_id: input_datetime.last_motion_notification
        data:
          datetime: "{{ now().isoformat() }}"
      - action: notify.mobile_app_tonys_iphone
        data:
          message: "Motion detected"
```

## Escalating Notifications

Escalate if nobody responds:

```yaml
automation:
  - alias: "Escalating notification"
    id: escalating_notification
    triggers:
      - trigger: state
        entity_id: binary_sensor.water_leak
        to: "on"
    actions:
      # Step 1: Normal notification
      - action: notify.mobile_app_tonys_iphone
        data:
          title: "Water leak"
          message: "Leak detected!"
          data:
            tag: "water_leak"
            actions:
              - action: "LEAK_HANDLED"
                title: "Handled"

      # Wait 5 minutes
      - wait_for_trigger:
          - trigger: event
            event_type: mobile_app_notification_action
            event_data:
              action: "LEAK_HANDLED"
        timeout:
          minutes: 5

      # Step 2: Critical if nobody responded
      - if:
          - condition: template
            value_template: "{{ wait.trigger is none }}"
        then:
          - action: notify.all_phones
            data:
              title: "⚠️ WATER LEAK - NOT ACKNOWLEDGED"
              message: "Leak still active!"
              data:
                push:
                  sound:
                    critical: 1
                actions:
                  - action: "LEAK_HANDLED"
                    title: "Handled"

      # Step 3: Automatic shutoff
      - wait_for_trigger:
          - trigger: event
            event_type: mobile_app_notification_action
            event_data:
              action: "LEAK_HANDLED"
        timeout:
          minutes: 5

      - if:
          - condition: template
            value_template: "{{ wait.trigger is none }}"
        then:
          - action: switch.turn_off
            target:
              entity_id: switch.water_main
          - action: notify.all_phones
            data:
              title: "Water shut off"
              message: "The water supply was shut off automatically"
```

## Best Practices

1. **Use groups** - `notify.all_phones` instead of multiple services
2. **Tag notifications** - So they can be updated/removed
3. **Rate limit** - Avoid notification spam
4. **Respect focus modes** - Check before non-critical notifications
5. **Escalate important events** - Critical events require acknowledgment
6. **Group related notifications** - Use `group` to collect similar ones
7. **Actionable** - Add buttons for common actions
