# Presence Detection

Guide to reliable presence detection in Home Assistant.

## Overview

Presence detection is the foundation of smart home automation. Combining several methods gives the best result.

## Detection Methods

| Method | Room-level | Home-level | Latency | Reliability |
|--------|------------|------------|---------|-------------|
| PIR sensor | ✅ | ❌ | <1s | High (motion) |
| mmWave radar | ✅ | ❌ | <1s | Very high |
| BLE | ✅ | ✅ | 1-30s | Medium |
| WiFi tracking | ❌ | ✅ | 1-60s | Medium |
| GPS (phone) | ❌ | ✅ | 1-5min | High (outdoors) |
| Router integration | ❌ | ✅ | 1-5min | Medium |

## PIR Motion Sensor

Basic motion detection:

```yaml
# Generated by aurora@aurora-smart-home (home-assistant skill) v1.7.9
# https://github.com/tonylofgren/aurora-smart-home

automation:
  - alias: "Living Room - Light on motion"
    id: living_room_motion_light
    mode: restart
    triggers:
      - trigger: state
        entity_id: binary_sensor.living_room_motion
        to: "on"
    conditions:
      - condition: numeric_state
        entity_id: sensor.living_room_illuminance
        below: 50
    actions:
      - action: light.turn_on
        target:
          entity_id: light.living_room
        data:
          brightness_pct: 80
      - wait_for_trigger:
          - trigger: state
            entity_id: binary_sensor.living_room_motion
            to: "off"
            for:
              minutes: 5
        timeout:
          minutes: 30
      - action: light.turn_off
        target:
          entity_id: light.living_room
```

## mmWave Radar (Presence, Not Motion)

Detects people sitting still:

```yaml
# ESPHome config - see esphome/references/sensors.md

# HA Automation
automation:
  - alias: "Office - Light on presence"
    id: office_presence_light
    triggers:
      - trigger: state
        entity_id: binary_sensor.office_presence_mmwave
        to: "on"
    actions:
      - action: light.turn_on
        target:
          entity_id: light.office

  - alias: "Office - Light off on absence"
    id: office_absence_light
    triggers:
      - trigger: state
        entity_id: binary_sensor.office_presence_mmwave
        to: "off"
        for:
          minutes: 2  # Short timeout, mmWave is reliable
    actions:
      - action: light.turn_off
        target:
          entity_id: light.office
```

## Bayesian Sensor

Combine multiple signals with probability:

```yaml
# configuration.yaml
binary_sensor:
  - platform: bayesian
    name: "Tony Home"
    unique_id: tony_home_bayesian
    prior: 0.5  # 50% chance of being home
    probability_threshold: 0.9
    observations:
      # GPS - High reliability
      - platform: state
        entity_id: person.tony
        to_state: "home"
        prob_given_true: 0.99
        prob_given_false: 0.05

      # WiFi - Medium reliability
      - platform: state
        entity_id: device_tracker.tonys_iphone
        to_state: "home"
        prob_given_true: 0.95
        prob_given_false: 0.10

      # BLE - Low reliability (can miss)
      - platform: state
        entity_id: device_tracker.tonys_iphone_ble
        to_state: "home"
        prob_given_true: 0.80
        prob_given_false: 0.20

      # Router - Medium reliability
      - platform: state
        entity_id: device_tracker.tony_router
        to_state: "home"
        prob_given_true: 0.90
        prob_given_false: 0.15
```

## Group Sensor

Simpler alternative to Bayesian:

```yaml
# configuration.yaml
binary_sensor:
  - platform: group
    name: "Anyone Home"
    unique_id: anyone_home
    device_class: presence
    entities:
      - person.tony
      - person.anna
    all: false  # At least one must be home
```

## Zone-Based Automation

```yaml
# configuration.yaml
zone:
  - name: Work
    latitude: 59.3293
    longitude: 18.0686
    radius: 100
    icon: mdi:briefcase

  - name: Preschool
    latitude: 59.3200
    longitude: 18.0750
    radius: 50
    icon: mdi:baby

automation:
  - alias: "Leaving work - Preheat house"
    id: leaving_work_preheat
    triggers:
      - trigger: zone
        entity_id: person.tony
        zone: zone.work
        event: leave
    conditions:
      - condition: time
        after: "15:00:00"
        before: "19:00:00"
    actions:
      - action: climate.set_temperature
        target:
          entity_id: climate.living_room
        data:
          temperature: 22

  - alias: "Everyone gone - Energy saving mode"
    id: away_mode
    triggers:
      - trigger: state
        entity_id: binary_sensor.anyone_home
        to: "off"
        for:
          minutes: 10
    actions:
      - action: script.turn_on
        target:
          entity_id: script.away_mode
```

## BLE Room Presence

Detect which room someone is in:

```yaml
# With ESPHome BLE proxies in each room
# See esphome/references/ble-proxy.md

template:
  - sensor:
      - name: "Tony Room"
        unique_id: tony_room_location
        state: >
          {% set rooms = {
            'binary_sensor.living_room_tony_ble': 'Living Room',
            'binary_sensor.bedroom_tony_ble': 'Bedroom',
            'binary_sensor.kitchen_tony_ble': 'Kitchen',
            'binary_sensor.office_tony_ble': 'Office'
          } %}
          {% for sensor, room in rooms.items() %}
            {% if is_state(sensor, 'on') %}
              {{ room }}
            {% endif %}
          {% endfor %}
          {% if this.state is undefined %}
            Unknown
          {% endif %}
```

## Away Mode Automation

Complete away/home detection:

```yaml
# Generated by aurora@aurora-smart-home (home-assistant skill) v1.7.9
# https://github.com/tonylofgren/aurora-smart-home

automation:
  # Someone arrives home
  - alias: "Home - Welcome lights"
    id: arriving_home_lights
    triggers:
      - trigger: state
        entity_id: binary_sensor.anyone_home
        from: "off"
        to: "on"
    conditions:
      - condition: sun
        after: sunset
        after_offset: "-00:30:00"
    actions:
      - action: light.turn_on
        target:
          entity_id:
            - light.hallway
            - light.living_room
        data:
          brightness_pct: 80
          transition: 5

  # Everyone leaves
  - alias: "Away - Turn everything off"
    id: everyone_left
    triggers:
      - trigger: state
        entity_id: binary_sensor.anyone_home
        to: "off"
        for:
          minutes: 15  # Buffer to avoid false positives
    actions:
      - parallel:
          - action: light.turn_off
            target:
              entity_id: all
          - action: climate.set_preset_mode
            target:
              entity_id: climate.all
            data:
              preset_mode: away
          - action: media_player.turn_off
            target:
              entity_id: all
      # Confirmation
      - action: notify.mobile_app_tonys_iphone
        data:
          title: "House closed"
          message: "Everyone has left. Energy saving mode activated."

  # Long absence - Extra measures
  - alias: "Away long - Vacation mode"
    id: vacation_mode
    triggers:
      - trigger: state
        entity_id: binary_sensor.anyone_home
        to: "off"
        for:
          hours: 24
    actions:
      - action: water_heater.set_operation_mode
        target:
          entity_id: water_heater.boiler
        data:
          operation_mode: "eco"
      - action: input_boolean.turn_on
        target:
          entity_id: input_boolean.vacation_mode
```

## Room-Specific Presence

```yaml
# Generated by aurora@aurora-smart-home (home-assistant skill) v1.7.9
# https://github.com/tonylofgren/aurora-smart-home

template:
  - binary_sensor:
      - name: "Living Room Occupied"
        unique_id: living_room_occupied
        device_class: occupancy
        state: >
          {{
            is_state('binary_sensor.living_room_motion', 'on') or
            is_state('binary_sensor.living_room_presence', 'on') or
            is_state('media_player.living_room_tv', 'playing')
          }}
        delay_off:
          minutes: 10

      - name: "Bedroom Occupied"
        unique_id: bedroom_occupied
        device_class: occupancy
        state: >
          {{
            is_state('binary_sensor.bedroom_motion', 'on') or
            is_state('binary_sensor.bedroom_presence', 'on') or
            (now().hour >= 22 or now().hour < 7)
            and is_state('person.tony', 'home')
          }}
```

### Common Problems

| Problem | Cause | Solution |
|---------|-------|----------|
| False "away" | WiFi sleep | Increase consider_home |
| Slow detection | GPS interval | Reduce interval in app |
| BLE unreliable | Signal strength | More BLE proxies |
| Router misses | DHCP lease | Static IPs |

## Best Practices

1. **Combine methods** - Use at least 2-3 different signals
2. **Bayesian for home-level** - Probability-based is most reliable
3. **mmWave for room-level** - Best for people sitting still
4. **Buffer for away** - At least 10-15 min before "away" activates
5. **Notifications** - Confirm critical actions
6. **consider_home** - Increase for unreliable trackers
