# Calendar Automation

Guide for automating based on calendar data in Home Assistant.

## Overview

Calendar automation enables:
- Scheduling via calendar instead of YAML
- Integration with Google Calendar, CalDAV, etc.
- Dynamic schedules that are easy to change
- Vacation and public holiday handling

## Calendar Platforms

| Platform | Description | Local |
|----------|-------------|-------|
| Local Calendar | Built-in HA calendar | Yes |
| Google Calendar | Google integration | No |
| CalDAV | iCloud, Nextcloud, etc. | Depends |
| ICS | iCal files | Yes |
| Waste Collection | Waste pickup schedule | Yes |

## Local Calendar

### Create Local Calendar

Settings → Devices & Services → Add Integration → Local Calendar

```yaml
# Automatically created entity:
# calendar.my_schedule
```

### Add Event via Service

```yaml
# Generated by aurora@aurora-smart-home (home-assistant skill) v1.7.9
# https://github.com/tonylofgren/aurora-smart-home

automation:
  - alias: "Add vacation"
    id: add_vacation_event
    triggers:
      - trigger: state
        entity_id: input_boolean.vacation_mode
        to: "on"
    actions:
      - action: calendar.create_event
        target:
          entity_id: calendar.vacation
        data:
          summary: "Vacation"
          start_date: "{{ now().date().isoformat() }}"
          end_date: "{{ (now() + timedelta(days=7)).date().isoformat() }}"
```

## Calendar Trigger

### Basic Calendar Trigger

```yaml
automation:
  - alias: "Calendar - Event starts"
    id: calendar_event_start
    triggers:
      - trigger: calendar
        entity_id: calendar.arbetsschema
        event: start
    actions:
      - action: switch.turn_on
        target:
          entity_id: switch.office_lights
      - action: climate.set_temperature
        target:
          entity_id: climate.office
        data:
          temperature: 22
```

### Event Ends

```yaml
automation:
  - alias: "Calendar - Event ends"
    id: calendar_event_end
    triggers:
      - trigger: calendar
        entity_id: calendar.arbetsschema
        event: end
    actions:
      - action: switch.turn_off
        target:
          entity_id: switch.office_lights
      - action: climate.set_preset_mode
        target:
          entity_id: climate.office
        data:
          preset_mode: away
```

### Offset Trigger

Trigger before/after an event:

```yaml
automation:
  - alias: "Calendar - 30 min before meeting"
    id: calendar_before_meeting
    triggers:
      - trigger: calendar
        entity_id: calendar.work
        event: start
        offset: "-00:30:00"  # 30 minutes before
    conditions:
      - condition: template
        value_template: "{{ 'meeting' in trigger.calendar_event.summary.lower() }}"
    actions:
      - action: notify.mobile_app_tonys_iphone
        data:
          title: "Meeting soon"
          message: "{{ trigger.calendar_event.summary }} starts in 30 minutes"
```

## Trigger Data

Available data in `trigger.calendar_event`:

```yaml
automation:
  - alias: "Calendar - Show event data"
    id: calendar_event_data
    triggers:
      - trigger: calendar
        entity_id: calendar.home
        event: start
    actions:
      - action: notify.mobile_app_tonys_iphone
        data:
          title: "{{ trigger.calendar_event.summary }}"
          message: >
            Start: {{ trigger.calendar_event.start }}
            End: {{ trigger.calendar_event.end }}
            Description: {{ trigger.calendar_event.description | default('None') }}
            Location: {{ trigger.calendar_event.location | default('None') }}
```

## Condition Checks with Calendar

### State-based

```yaml
automation:
  - alias: "Only during work hours"
    id: only_work_hours
    triggers:
      - trigger: state
        entity_id: binary_sensor.motion
        to: "on"
    conditions:
      # Check if there is an active event
      - condition: state
        entity_id: calendar.work_schedule
        state: "on"
    actions:
      - action: light.turn_on
        target:
          entity_id: light.office
```

### Template-based

```yaml
automation:
  - alias: "Check specific event"
    id: check_specific_event
    triggers:
      - trigger: time_pattern
        minutes: "/30"
    conditions:
      - condition: template
        value_template: >
          {% set cal = state_attr('calendar.work_schedule', 'message') %}
          {{ cal is not none and 'remote work' in cal.lower() }}
    actions:
      - action: light.turn_on
        target:
          entity_id: light.office
```

## Vacation Mode

### Automatic Vacation from Calendar

```yaml
automation:
  - alias: "Vacation - Activate"
    id: vacation_start
    triggers:
      - trigger: calendar
        entity_id: calendar.vacation
        event: start

          entity_id: all
        data:
          preset_mode: away

    id: vacation_end

        target:
          entity_id: input_boolean.vacation_mode
      - action: climate.set_preset_mode
        target:

        data:
          preset_mode: home
```

## Work Schedule

### Different Schedules per Day

```yaml
# Create events in the calendar:
# - "Morning" 06:00-08:00 (Monday-Friday)
# - "Work hours" 08:00-17:00 (Monday-Friday)
# - "Evening" 17:00-22:00 (every day)
# - "Night" 22:00-06:00 (every day)

automation:
  - alias: "Schedule - Morning routine"
    id: schedule_morning
    triggers:
      - trigger: calendar
        entity_id: calendar.daily_schedule
        event: start
    conditions:
      - condition: template
        value_template: "{{ trigger.calendar_event.summary == 'Morning' }}"
    actions:
      - action: scene.turn_on
        target:
          entity_id: scene.morning

  - alias: "Schedule - Night mode"
    id: schedule_night
    triggers:
      - trigger: calendar
        entity_id: calendar.daily_schedule
        event: start
    conditions:
      - condition: template
        value_template: "{{ trigger.calendar_event.summary == 'Night' }}"
    actions:
      - action: scene.turn_on
        target:
          entity_id: scene.night
```

## Dynamic Schedules

### Fetch Upcoming Events

```yaml
template:
  - sensor:
      - name: "Next calendar event"
        unique_id: next_calendar_event
        state: "{{ state_attr('calendar.home', 'message') | default('None') }}"
        attributes:
          start: "{{ state_attr('calendar.home', 'start_time') }}"
          end: "{{ state_attr('calendar.home', 'end_time') }}"
          location: "{{ state_attr('calendar.home', 'location') }}"
          description: "{{ state_attr('calendar.home', 'description') }}"
```

### Automatic Lighting Based on Event

```yaml
automation:
  - alias: "Dynamic lighting per event"
    id: dynamic_lighting_by_event
    triggers:
      - trigger: calendar
        entity_id: calendar.home
        event: start
    actions:
      - choose:
          # Movie night
          - conditions:
              - condition: template
                value_template: "{{ 'movie' in trigger.calendar_event.summary.lower() }}"
            sequence:
              - action: scene.turn_on
                target:
                  entity_id: scene.movie

          # Dinner
          - conditions:
              - condition: template
                value_template: "{{ 'dinner' in trigger.calendar_event.summary.lower() }}"
            sequence:
              - action: scene.turn_on
                target:
                  entity_id: scene.dinner

          # Party
          - conditions:
              - condition: template
                value_template: "{{ 'party' in trigger.calendar_event.summary.lower() }}"
            sequence:
              - action: scene.turn_on
                target:
                  entity_id: scene.party
```

```yaml
  - alias: "Work meetings only"
    id: work_meetings_only
    triggers:
      - trigger: calendar
        entity_id: calendar.work_calendar
        event: start
```

## Workday Sensor

Combine with the workday sensor:

```yaml
# configuration.yaml
binary_sensor:
  - platform: workday
    country: SE
    workdays: [mon, tue, wed, thu, fri]
    excludes: [sat, sun, holiday]
```

```yaml
automation:
  - alias: "Workday morning"
    id: workday_morning
    triggers:
      - trigger: time
        at: "07:00:00"
    conditions:
      - condition: state
        entity_id: binary_sensor.workday_sensor
        state: "on"
      # Not on vacation
      - condition: state
        entity_id: calendar.vacation
        state: "off"
    actions:
      - action: scene.turn_on
        target:
          entity_id: scene.workday_morning
```

### Customize Automations for Public Holidays

```yaml
automation:
  - alias: "Holiday - Sleep in"
    id: holiday_sleep_in
    triggers:
      - trigger: time
        at: "09:00:00"  # Later than usual
    conditions:
      - condition: or
        conditions:
          # Saturday or Sunday
          - condition: time
            weekday:
              - sat
              - sun
          # Or public holiday
          - condition: state
            entity_id: binary_sensor.workday_sensor
            state: "off"
    actions:
      - action: scene.turn_on
        target:
          entity_id: scene.weekend_morning
```

## Meeting Room Booking

```yaml
automation:
  - alias: "Meeting room - Prepare"
    id: meeting_room_prepare
    triggers:
      - trigger: calendar
        entity_id: calendar.meeting_room
        event: start
        offset: "-00:05:00"
    actions:
      - parallel:

              entity_id: light.meeting_room

              brightness_pct: 100

              entity_id: climate.meeting_room
```

## Best Practices

1. **Use Local Calendar** for schedules you control
2. **Offset triggers** for preparation/cleanup
3. **Combine with workday** for public holiday handling
4. **Filter events** with template conditions
5. **Backup** - Export calendar regularly
6. **Naming convention** - Consistent event names for filtering
